# Apéndice

Este capítulo proporciona materiales de referencia y normas de desarrollo para OpenCode Agent Skills, ayudándote a escribir skills de alta calidad y consultar rápidamente las interfaces de API.

## Contenido de este Capítulo

| Página | Descripción |
|---|---|
| [Mejores Prácticas de Desarrollo de Skills](./best-practices/) | Domina las técnicas y normas para escribir skills de alta calidad, incluyendo convenciones de nombres, redacción de descripciones, organización de directorios y uso de scripts |
| [Referencia de Herramientas API](./api-reference/) | Consulta los parámetros detallados y valores de retorno de todas las herramientas disponibles, incluyendo `get_available_skills`, `read_skill_file`, `run_skill_script`, `use_skill` |

## Sugerencias de Ruta de Aprendizaje

::: tip Orden Recomendado
1. **Primero las Mejores Prácticas** → Entiende las normas y técnicas del desarrollo de skills para evitar errores comunes
2. **Luego la Referencia API** → Consulta parámetros y valores de retorno específicos cuando surjan problemas durante el desarrollo real
:::

Estos dos contenidos tienen diferentes propósitos:

- **Mejores Prácticas**: Adecuado para leer completamente antes de comenzar a escribir skills, estableciendo hábitos correctos de desarrollo
- **Referencia API**: Adecuado como libro de referencia, consultando detalles específicos cuando sea necesario

## Condiciones Previas

::: warning Recomendado Antes
Antes de leer este capítulo, se recomienda que ya hayas:
- Completado [Crear tu Primer Skill](../start/creating-your-first-skill/), entendiendo la estructura básica de skills
- Entendido el [Mecanismo de Descubrimiento de Skills](../platforms/skill-discovery-mechanism/), comprendiendo cómo el plugin encuentra skills
:::
