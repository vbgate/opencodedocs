# Funciones de Plataforma

Esta sección introduce las dos funciones principales de Plannotator: **Revisión de Planes** y **Revisión de Código**. Aprenderás a revisar visualmente los planes generados por IA, agregar varios tipos de anotaciones, adjuntar marcadores de imagen y revisar los cambios de código git diff.

## Prerrequisitos

::: warning Confirma antes de comenzar
Antes de estudiar esta sección, asegúrate de haber completado los siguientes preparativos:

- ✅ Completado el tutorial de [Inicio Rápido](../start/getting-started/)
- ✅ Instalado el plugin de Plannotator ([Claude Code](../start/installation-claude-code/) u [OpenCode](../start/installation-opencode/))
:::

## Contenido de Esta Sección

### Revisión de Planes

Aprende a revisar los planes de ejecución generados por IA, agregar sugerencias de modificación y hacer que la IA ejecute según tus intenciones.

| Tutorial | Descripción |
| --- | ---|
| [Fundamentos de Revisión de Planes](./plan-review-basics/) | Aprende a usar Plannotator para revisar visualmente los planes generados por IA, incluyendo aprobar o rechazar planes |
| [Agregar Anotaciones al Plan](./plan-review-annotations/) | Domina cómo agregar diferentes tipos de anotaciones al plan (eliminar, reemplazar, insertar, comentar) |
| [Agregar Marcadores de Imagen](./plan-review-images/) | Aprende a adjuntar imágenes en la revisión de planes y usar herramientas de pincel, flecha y círculo para marcar |

### Revisión de Código

Aprende a revisar cambios de código, agregar comentarios a nivel de línea y descubrir problemas antes de confirmar.

| Tutorial | Descripción |
| --- | ---|
| [Agregar Comentarios de Código](./code-review-annotations/) | Domina cómo agregar comentarios a nivel de línea en la revisión de código (comment/suggestion/concern) |
| [Cambiar Vistas de Diff](./code-review-diff-types/) | Aprende a cambiar entre diferentes tipos de diff en la revisión de código (uncommitted/staged/last commit/branch) |

## Rutas de Aprendizaje

::: tip Orden de Aprendizaje Recomendado
Según tu caso de uso, elige la ruta de aprendizaje adecuada:

**Ruta A: Revisión de Planes Primero** (recomendado para principiantes)
1. [Fundamentos de Revisión de Planes](./plan-review-basics/) → Aprende primero el flujo básico de revisión de planes
2. [Agregar Anotaciones al Plan](./plan-review-annotations/) → Aprende a modificar planes con precisión
3. [Agregar Marcadores de Imagen](./plan-review-images/) → Usa imágenes para expresar intenciones más claramente
4. Luego aprende la serie de revisión de código

**Ruta B: Revisión de Código Primero** (adecuado para desarrolladores con experiencia en Code Review)
1. [Fundamentos de Revisión de Código](./code-review-basics/) → Familiarízate con la interfaz de revisión de código
2. [Agregar Comentarios de Código](./code-review-annotations/) → Aprende comentarios a nivel de línea
3. [Cambiar Vistas de Diff](./code-review-diff-types/) → Domina diferentes tipos de diff
4. Luego aprende la serie de revisión de planes
:::
