# Escribir reglas de mejores prácticas de React

## Lo que podrás hacer al terminar

- Escribir desde cero reglas de optimización de rendimiento de React que cumplan con especificaciones de Agent Skills
- Usar la plantilla _template.md para crear rápidamente archivos de reglas
- Seleccionar correctamente el nivel de impacto (CRITICAL/HIGH/MEDIUM) para clasificar reglas
- Escribir ejemplos de comparación de código Incorrecto/Correcto claros y fáciles de entender
- Verificar la integridad de las reglas con `pnpm validate`

## Tu situación actual

Ya has usado la biblioteca de reglas de optimización de rendimiento de React de Agent Skills para auditar código y has descubierto que algunos patrones de optimización de rendimiento no están cubiertos. Quieres contribuir con tus propias reglas, pero:

- No sabes qué formato deben tener los archivos de reglas
- No estás seguro de cómo seleccionar el nivel de impacto
- Los ejemplos de código que escribes no cumplen con las especificaciones, lo que provoca fallos de verificación
- Temes que tus reglas sean rechazadas o no puedan analizarse correctamente

## Cuándo usar esta técnica

Escenarios adecuados para escribir reglas:

- Descubriste problemas de rendimiento en tu proyecto y quieres solidificarlos como reglas reutilizables
- El equipo tiene patrones específicos de optimización de React y quieres que el Agente los aplique
- Aprendiste nuevos trucos de la documentación oficial de React o del equipo de ingeniería de Vercel y quieres compartirlos con la comunidad
