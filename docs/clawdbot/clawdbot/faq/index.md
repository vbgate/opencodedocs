# FAQ and Optimization

This chapter helps you solve various issues when using Clawdbot, master complete command-line operations, and learn how to optimize system performance.

::: info Recommended Learning Order
It is recommended to read this chapter in the following order:
1. First learn **[CLI Command Reference](./cli-commands/)** —— Master basic operations and debugging tools
2. Then learn **[Troubleshooting](./troubleshooting/)** —— Quickly locate and solve problems when they occur
3. Finally learn **[Performance Optimization](./performance-optimization/)** —— Improve system efficiency
:::

## Subpage Navigation

### [Troubleshooting](./troubleshooting/)

A comprehensive troubleshooting guide for common issues, covering:
- Gateway startup failures
- Channel connection issues (WhatsApp, Telegram, Slack, etc.)
- Authentication errors and permission problems
- Tool invocation failures
- Session management exceptions

**When to use**: When you encounter specific problems and need to quickly troubleshoot and solve them.

---

### [CLI Command Reference](./cli-commands/)

Complete command list and usage instructions, including:
- Gateway control commands (start, restart, status)
- Agent interaction commands
- Model management and configuration
- Channel management (login, status, logout)
- Session management (list, reset, delete, compress)
- Node management (list, invoke, pair)
- Skill management and configuration wizard

**When to use**: When you need to quickly find command usage and parameters, or learn the complete CLI functionality.

---

### [Performance Optimization](./performance-optimization/)

Optimization techniques to improve Gateway performance and response speed, covering:
- Prompt Caching to reduce costs
- Session Pruning to reduce context
- Concurrency control and resource management
- Memory optimization and garbage collection
- Log level tuning

**When to use**: When the system is running slowly, resource usage is high, or you want to improve user experience.
