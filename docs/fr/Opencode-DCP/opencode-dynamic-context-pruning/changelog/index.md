# Journal des modifications

Ce chapitre enregistre l'historique de l'évolution des versions du plugin DCP, vous aidant à comprendre les nouvelles fonctionnalités, améliorations et corrections de chaque version afin de prendre des décisions de mise à niveau.

## Contenu de ce chapitre

| Page | Description |
|--- | ---|
| [Historique des versions](./version-history/) | Consultez l'historique complet des mises à jour de v1.0.1 à la dernière version, y compris les nouvelles fonctionnalités, les améliorations et les corrections de bugs |

## Recommandations de lecture

Le journal des modifications est principalement utilisé dans les scénarios suivants :

1. **Avant la mise à niveau** - Comprendre les changements apportés par la nouvelle version et évaluer si une mise à niveau est nécessaire
2. **Résolution de problèmes** - Identifier la version d'introduction d'une fonctionnalité ou la version de correction d'un bug
3. **Comprendre l'orientation du développement** - Apprendre l'évolution et les priorités d'optimisation de DCP à travers l'historique des versions

::: tip Garder à jour
Recommande d'utiliser l'étiquette `@latest` dans la configuration OpenCode pour obtenir automatiquement les dernières fonctionnalités et corrections.
:::
