# Annexe

Ce chapitre fournit des références et des spécifications de développement pour les OpenCode Agent Skills, vous aidant à écrire des skills de haute qualité et à consulter rapidement les interfaces API.

## Contenu de ce chapitre

| Page | Description |
| --- | ---|
| [Meilleures pratiques de développement de skills](./best-practices/) | Maîtrisez les techniques et spécifications pour écrire des skills de haute qualité, incluant les conventions de nommage, la rédaction de descriptions, l'organisation des répertoires, l'utilisation des scripts, etc. |
| [Référence des outils API](./api-reference/) | Consultez les paramètres détaillés et les valeurs de retour de tous les outils disponibles, incluant `get_available_skills`, `read_skill_file`, `run_skill_script`, `use_skill` |

## Recommandations de parcours d'apprentissage

::: tip Ordre recommandé
1. **Commencez par les meilleures pratiques** → Découvrez les spécifications et techniques de développement de skills pour éviter les erreurs courantes
2. **Consultez ensuite la référence API** → Lorsque vous rencontrez des problèmes en développement réel, consultez les paramètres spécifiques et les valeurs de retour
:::

Le positionnement de ces deux contenus est différent :

- **Meilleures pratiques** : Convient d'être lu entièrement avant de commencer à écrire des skills, pour établir de bonnes habitudes de développement
- **Référence API** : Convient comme outil de référence, à consulter pour des détails spécifiques en cas de besoin

## Prérequis

::: warning Suggéré avant lecture
Avant de lire ce chapitre, il est conseillé que vous ayez :
- Complété [Créer votre premier skill](../start/creating-your-first-skill/), comprendre la structure de base des skills
- Connaissance du [Mécanisme de découverte des skills](../platforms/skill-discovery-mechanism/), comprendre comment les plugins trouvent les skills
:::
