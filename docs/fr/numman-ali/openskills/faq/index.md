# FAQ

Cette section rassemble les questions fréquentes, les solutions de dépannage et les explications de la philosophie de conception rencontrées lors de l'utilisation d'OpenSkills, vous aidant à localiser et résoudre rapidement les problèmes.

## Contenu de cette section

<div class="grid-cards">

### [Questions fréquentes](./faq/)

Réponses aux questions courantes concernant l'installation, la configuration et l'utilisation. Y compris les différences entre OpenSkills et Claude Code, les emplacements d'installation des compétences, les problèmes de synchronisation d'AGENTS.md, le partage de compétences entre plusieurs projets, etc.

### [Dépannage](./troubleshooting/)

Fournit des méthodes de diagnostic et de réparation pour les erreurs courantes. Couvre les étapes de solution détaillées pour les échecs de Git clone, SKILL.md introuvable, compétences non trouvées, erreurs de permission, saut de mise à jour, et autres problèmes.

### [Pourquoi CLI ?](./why-cli-not-mcp/)

Explique la justification conceptuelle d'OpenSkills pour choisir CLI plutôt que MCP. Compare en profondeur les différences de positionnement entre les deux et comprend pourquoi les systèmes de compétences sont mieux adaptés au mode de fichiers statiques.

</div>

## Parcours d'apprentissage

```
Vous rencontrez un problème ?
    │
    ├─ Vous ne savez pas comment l'utiliser ───────→ Questions fréquentes
    │                                            (concepts, configuration, astuces d'utilisation)
    │
    ├─ Vous avez une erreur ──────────────────────→ Dépannage
    │                                            (diagnostic d'erreur, étapes de réparation)
    │
    └─ Vous voulez comprendre la conception ───────→ Pourquoi CLI ?
                                                 (choix technologiques, décisions d'architecture)
```

**Ordre recommandé** :

1. **Lisez d'abord les questions fréquentes** → Comprenez rapidement les concepts de base et les astuces d'utilisation
2. **Lisez le dépannage lorsque vous rencontrez des erreurs** → Trouvez des solutions par message d'erreur
3. **Lisez la philosophie de conception pour une compréhension approfondie** → Comprenez les considérations techniques de CLI vs MCP
