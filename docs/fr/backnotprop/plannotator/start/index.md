# Démarrage rapide

Ce chapitre vous aide à utiliser Plannotator à partir de zéro. Vous découvrirez ce qu'est Plannotator, les problèmes qu'il peut résoudre, et vous configurerez l'installation dans l'AI Coding Agent que vous utilisez (Claude Code ou OpenCode).

## Contenu de ce chapitre

<div class="grid-cards">

### [Démarrage rapide](./getting-started/)

Découvrez les fonctionnalités clés et les cas d'utilisation de Plannotator, et réalisez votre première expérience en 5 minutes. Convient à tous les nouveaux utilisateurs comme première leçon.

### [Installer le plugin Claude Code](./installation-claude-code/)

Installez et configurez le plugin Plannotator dans Claude Code. Prend en charge deux méthodes: le système de plugins et le hook manuel, couvrant les configurations locales et distantes.

### [Installer le plugin OpenCode](./installation-opencode/)

Installez et configurez le plugin Plannotator dans OpenCode, en activant l'outil `submit_plan` et la commande `/plannotator-review`.

</div>

## Parcours d'apprentissage

::: tip Ordre recommandé
1. **Lisez d'abord [Démarrage rapide](./getting-started/)** : Comprenez ce qu'est Plannotator, ce qu'il peut faire, et établissez une compréhension globale
2. **Choisissez ensuite un tutoriel d'installation** : Sélectionnez le guide d'installation correspondant à l'AI Coding Agent que vous utilisez
   - Utilisation de Claude Code → [Installer le plugin Claude Code](./installation-claude-code/)
   - Utilisation d'OpenCode → [Installer le plugin OpenCode](./installation-opencode/)
:::

Une fois l'installation terminée, vous pouvez commencer à utiliser Plannotator pour les revues de plan et les revues de code.

## Prochaines étapes

Après avoir terminé ce chapitre, continuez à apprendre les **[Fonctionnalités de la plateforme](../platforms/)** pour maîtriser l'utilisation complète des revues de plan et des revues de code:

- Bases de la revue de plan: approuver, rejeter, ajouter des commentaires
- Annotation d'images: joindre des captures d'écran dans les plans et les annoter
- Revue de code: visualiser les diffs Git et ajouter des commentaires au niveau des lignes
