# Annexe

Ce chapitre contient la documentation de référence et des documents supplémentaires pour vous aider à approfondir votre compréhension des détails techniques d'AI App Factory.

## Navigation des sous-pages

### [Référence des commandes CLI](../cli-commands/)
Liste complète des commandes CLI, descriptions des paramètres et exemples d'utilisation, incluant les commandes `init`, `run`, `continue`, `status`, `list` et `reset`.

### [Structure des répertoires](../directory-structure/)
Comprendre la structure complète des répertoires du projet Factory et l'usage de chaque fichier, y compris le rôle des répertoires principaux comme `agents/`, `skills/`, `cli/`.

### [Normes de code](../code-standards/)
Comprendre les normes de codage TypeScript, la structure des fichiers, les exigences de commentaires et les spécifications des messages de commit que le code généré doit respecter.

### [Pile technologique](../tech-stack/)
Comprendre la pile technologique utilisée par les applications générées : Node.js + Express + Prisma + React Native, ainsi que les justifications des choix techniques pour chaque composant.

## Recommandations de parcours d'apprentissage

La documentation de ce chapitre constitue une **référence** et n'a pas besoin d'être lue dans l'ordre. Nous vous recommandons de consulter les sections correspondantes en fonction de vos besoins tout en suivant le tutoriel principal :

- Lorsque vous étudiez [Installation et configuration](../../start/installation/), vous pouvez consulter [Référence des commandes CLI](../cli-commands/)
- Lorsque vous étudiez [Initialisation d'un projet Factory](../../start/init-project/), vous pouvez consulter [Structure des répertoires](../directory-structure/)
- Lorsque vous étudiez les différentes étapes du pipeline, vous pouvez consulter [Pile technologique](../tech-stack/)
- Lorsque vous avez besoin de comprendre les standards de qualité du code généré, vous pouvez consulter [Normes de code](../code-standards/)
