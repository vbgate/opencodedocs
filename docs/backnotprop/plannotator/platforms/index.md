# Platform Features

This chapter introduces Plannotator's two core features: **Plan Review** and **Code Review**. You'll learn how to visually review AI-generated plans, add various types of comments, attach image annotations, and review git diff code changes.

## Prerequisites

::: warning Before You Begin
Before studying this chapter, ensure you have completed the following preparations:

- ✅ Completed [Quick Start](../start/getting-started/) tutorial
- ✅ Installed Plannotator plugin ([Claude Code](../start/installation-claude-code/) or [OpenCode](../start/installation-opencode/))
:::

## Learning Path

::: tip Recommended Learning Sequence
Choose the appropriate learning path based on your use case:

**Path A: Plan Review First** (Recommended for beginners)
1. [Plan Review Basics](./plan-review-basics/) → First learn the basic plan review process
2. [Add Plan Annotations](./plan-review-annotations/) → Learn how to precisely modify plans
3. [Add Image Annotations](./plan-review-images/) → Use images to express intent more clearly
4. Then learn the code review series

**Path B: Code Review First** (Suitable for developers with Code Review experience)
1. [Code Review Basics](./code-review-basics/) → Familiarize with code review interface
2. [Add Code Comments](./code-review-annotations/) → Learn line-level comments
3. [Switch Diff Views](./code-review-diff-types/) → Master different diff types
4. Then learn the plan review series
:::
