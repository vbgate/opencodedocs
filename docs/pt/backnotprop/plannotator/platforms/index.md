# Funcionalidades da Plataforma

Este capítulo apresenta as duas funcionalidades principais do Plannotator: **Revisão de Planos** e **Revisão de Código**. Você aprenderá como revisar visualmente os planos gerados por IA, adicionar vários tipos de anotações, anexar marcações de imagem e revisar alterações de código git diff.

## Pré-requisitos

::: warning Antes de começar
Antes de estudar este capítulo, certifique-se de ter concluído as seguintes preparações:

- ✅ Concluiu o tutorial de [Início Rápido](../start/getting-started/)
- ✅ Instalou o plugin Plannotator ([Claude Code](../start/installation-claude-code/) ou [OpenCode](../start/installation-opencode/))
:::

## Conteúdo deste Capítulo

### Revisão de Planos

Aprenda como revisar planos de execução gerados por IA, adicionar sugestões de modificação e fazer a IA executar de acordo com suas intenções.

| Tutorial | Descrição |
| --- | --- |
| [Fundamentos da Revisão de Planos](./plan-review-basics/) | Aprenda a usar o Plannotator para revisar visualmente planos gerados por IA, incluindo aprovar ou rejeitar planos |
| [Adicionar Anotações ao Plano](./plan-review-annotations/) | Domine como adicionar diferentes tipos de anotações ao plano (excluir, substituir, inserir, comentar) |
| [Adicionar Marcações de Imagem](./plan-review-images/) | Aprenda a anexar imagens na revisão de planos e usar ferramentas de pincel, seta e círculo para marcação |

### Revisão de Código

Aprenda como revisar alterações de código, adicionar anotações em nível de linha e identificar problemas antes do commit.

| Tutorial | Descrição |
| --- | --- |
| [Adicionar Anotações de Código](./code-review-annotations/) | Domine como adicionar anotações em nível de linha na revisão de código (comentário/sugestão/preocupação) |
| [Alternar Visualizações de Diff](./code-review-diff-types/) | Aprenda a alternar entre diferentes tipos de diff na revisão de código (uncommitted/staged/último commit/branch) |

## Caminho de Aprendizagem

::: tip Ordem de Aprendizagem Recomendada
De acordo com seu cenário de uso, escolha o caminho de aprendizagem apropriado:

**Caminho A: Revisão de Planos Prioritária** (Recomendado para iniciantes)
1. [Fundamentos da Revisão de Planos](./plan-review-basics/) → Aprenda primeiro o fluxo básico de revisão de planos
2. [Adicionar Anotações ao Plano](./plan-review-annotations/) → Aprenda como modificar planos com precisão
3. [Adicionar Marcações de Imagem](./plan-review-images/) → Use imagens para expressar intenções mais claramente
4. Depois aprenda a série de revisão de código

**Caminho B: Revisão de Código Prioritária** (Adequado para desenvolvedores com experiência em Code Review)
1. [Fundamentos da Revisão de Código](./code-review-basics/) → Familiarize-se com a interface de revisão de código
2. [Adicionar Anotações de Código](./code-review-annotations/) → Aprenda anotações em nível de linha
3. [Alternar Visualizações de Diff](./code-review-diff-types/) → Domine diferentes tipos de diff
4. Depois aprenda a série de revisão de planos
:::
