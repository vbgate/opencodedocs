# Perguntas Frequentes e Otimização

Este capítulo ajuda você a resolver vários problemas ao usar o Clawdbot, dominar métodos completos de operação de linha de comando e aprender como otimizar o desempenho do sistema.

::: info Recomendação de Ordem de Aprendizado
É recomendado ler este capítulo na seguinte ordem:
1. Primeiro aprenda **[Referência de Comandos CLI](./cli-commands/)** —— Domine operações básicas e ferramentas de depuração
2. Depois aprenda **[Solução de Problemas](./troubleshooting/)** —— Para localizar e resolver rapidamente problemas quando ocorrerem
3. Finalmente aprenda **[Otimização de Desempenho](./performance-optimization/)** —— Para melhorar a eficiência do sistema
:::

## Navegação de Subpáginas

### [Solução de Problemas](./troubleshooting/)

Guia completo de solução de problemas para problemas comuns, cobrindo:
- Falhas de inicialização do Gateway
- Problemas de conexão de canais (WhatsApp, Telegram, Slack, etc.)
- Erros de autenticação e problemas de permissão
- Falhas na invocação de ferramentas
- Exceções no gerenciamento de sessões

**Quando usar**: Quando você encontra problemas específicos e precisa solucioná-los e resolvê-los rapidamente.

---

### [Referência de Comandos CLI](./cli-commands/)

Lista completa de comandos e instruções de uso, incluindo:
- Comandos de controle do Gateway (início, reinicialização, status)
- Comandos de interação do Agent
- Gerenciamento e configuração de modelos
- Gerenciamento de canais (login, status, logout)
- Gerenciamento de sessões (lista, redefinição, exclusão, compactação)
- Gerenciamento de nós (lista, invocação, emparelhamento)
- Gerenciamento de habilidades e assistente de configuração

**Quando usar**: Quando você precisa encontrar rapidamente o uso e parâmetros de um comando, ou aprender a funcionalidade completa do CLI.

---

### [Otimização de Desempenho](./performance-optimization/)

Técnicas de otimização para melhorar o desempenho e a velocidade de resposta do Gateway, cobrindo:
- Cache de Prompts (Prompt Caching) para reduzir custos
- Poda de Sessões (Session Pruning) para reduzir contexto
- Controle de concorrência e gerenciamento de recursos
- Otimização de memória e coleta de lixo
- Ajuste de níveis de log

**Quando usar**: Quando o sistema está lento, o uso de recursos é alto, ou você deseja melhorar a experiência do usuário.
