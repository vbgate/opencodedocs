# 변경 로그

이 장에서는 Plannotator의 버전 이력, 신기능 릴리스 및 버그 수정 정보를 기록하여 프로젝트의 진화 과정과 최신 진행 상황을 파악할 수 있도록 도와드립니다.

## 읽기 권장 사항

변경 로그는 다음과 같은 시나리오에 적합합니다:

| 시나리오 | 권장 사항 |
|---|---|
| **업그레이드 전 확인** | 대상 버전의 신기능과 주요 변경 사항 확인 |
| **기능 탐색** | 최신 버전에 어떤 신기능이 추가되었는지 확인 |
| **문제 해결** | 특정 버그가 새 버전에서 수정되었는지 확인 |
| **버전 호환성** | Plannotator와 Claude Code/OpenCode의 버전 대응 관계 확인 |

::: tip 업데이트 유지
변경 로그를 정기적으로 확인하여 신기능과 보안 수정 사항을 얻을 수 있도록 최신 버전으로 즉시 업그레이드하는 것을 권장합니다.
:::
