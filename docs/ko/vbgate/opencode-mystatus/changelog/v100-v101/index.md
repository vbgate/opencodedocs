# v1.0.0 - v1.0.1: 초기 버전 릴리스 및 슬래시 명령 수정

## 버전 개요

**v1.0.0**(2026-01-11)은 opencode-mystatus의 초기 버전으로, 다중 플랫폼 할당량 조회의 핵심 기능을 가져왔습니다.

**v1.0.1**(2026-01-11)이 즉시 뒤따르며, 슬래시 명령 지원의 주요 문제를 수정했습니다.

---

## v1.0.1 - 슬래시 명령 수정

### 수정된 문제

**`command/` 디렉토리를 npm 패키지에 포함**

- **문제 설명**: v1.0.0 릴리스 후, 슬래시 명령 `/mystatus`가 정상적으로 작동하지 않음
- **원인 분석**: npm 패키징 시 `command/` 디렉토리가 누락되어 OpenCode가 슬래시 명령을 인식하지 못함
- **수정 방안**: `package.json`의 `files` 필드를 업데이트하여 `command/` 디렉토리가 릴리스 패키지에 포함되도록 함
- **영향 범위**: npm을 통해 설치한 사용자에게만 영향을 미치며, 수동 설치에는 영향이 없음

### 업그레이드 제안

이미 v1.0.0을 설치한 경우, 완전한 슬래시 명령 지원을 위해 즉시 v1.0.1로 업그레이드하는 것이 좋습니다:

```bash
## 최신 버전으로 업그레이드
npm update @vbgate/opencode-mystatus
```

## v1.0.0 - 초기 버전 릴리스

### 새로운 기능

**1. 다중 플랫폼 할당량 조회**

다음 플랫폼의 할당량 사용 현황을 한 번에 조회할 수 있습니다:

| 플랫폼 | 지원되는 구독 유형 | 할당량 유형 |
| ---- | -------------- | -------- |
| OpenAI | Plus/Team/Pro | 3시간 한도, 24시간 한도 |
| Zhipu AI | Coding Plan | 5시간 토큰 한도, MCP 월별 할당량 |
| Google Cloud | Antigravity | G3 Pro, G3 Image, G3 Flash, Claude |

**2. 시각화 진행률 막대**

할당량 사용 현황을 직관적으로 표시합니다:

```
OpenAI (user@example.com)
━━━━━━━━━━━━━━━━━━━━ 75%
사용됨 750 / 1000 요청
```

**3. 다국어 지원**

- 중국어(간체)
- 영어

언어 자동 감지, 수동 전환 불필요.

**4. API 키 보안 마스킹**

모든 민감 정보(API 키, OAuth 토큰)가 자동으로 마스킹됩니다:

```
Zhipu AI (zhipuai-coding-plan)
API 키: sk-a1b2****xyz
```

## 사용 방법

### 슬래시 명령(권장)

OpenCode에서 입력합니다:

```
/mystatus
```

### 자연어

다음과 같이 자연어로 질문할 수도 있습니다:

```
모든 AI 플랫폼 할당량 보기
```

---

## 업그레이드 가이드

### v1.0.0에서 v1.0.1로 업그레이드

```bash
npm update @vbgate/opencode-mystatus
```

업그레이드 후 OpenCode를 재시작하면 슬래시 명령 `/mystatus`를 사용할 수 있습니다.

### 첫 설치

```bash
npm install -g @vbgate/opencode-mystatus
```

설치 완료 후, OpenCode에서 `/mystatus`를 입력하면 모든 플랫폼의 할당량을 조회할 수 있습니다.

## 알려진 제한 사항

- v1.0.0은 GitHub Copilot을 지원하지 않습니다(v1.2.0에서 추가)
- v1.0.0은 Z.ai를 지원하지 않습니다(v1.1.0에서 추가)

이러한 기능을 사용하려면 최신 버전으로 업그레이드하십시오.
