# 변경 로그

이 장에서는 Antigravity Tools의 버전 변경 이력과 릴리스 설명을 기록합니다. 업그레이드 전 Changelog를 먼저 확인하면 호환성 변경 또는 회귀 문제에 빠질 확률이 줄어듭니다.

## 이 장의 내용

| 페이지 | 설명 |
|--- | ---|
| [버전 변경 이력](./release-notes/) | README 내장 Changelog를 읽는 방법, 업그레이드 영향을 빠르게 판단하고 업그레이드 후 검증 완료 |

## 학습 경로

1. **업그레이드 전**: Settings → About에서 현재 버전을 먼저 확인한 다음 README의 Changelog에서 건너뛴 버전 구간만 보기
2. **업그레이드 후**: `/healthz` 최소 검증을 한 번 실행하여 프록시가 정상 작동하는지 확인
