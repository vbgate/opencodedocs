# FAQ 및 최적화

이 장절은 Clawdbot 사용 시 발생하는 다양한 문제를 해결하고, 완전한 명령줄 작업 방법을 마스터하며, 시스템 성능을 최적화하는 방법을 배우는 데 도움을 줍니다.

::: info 학습 순서 권장
이 장절의 내용을 다음 순서로 읽는 것을 권장합니다:
1. 먼저 **[CLI 명령 참조](./cli-commands/)** 학습 —— 기본 작업 및 디버깅 도구 마스터
2. 다음으로 **[문제 해결](./troubleshooting/)** 학습 —— 문제 발생 시 빠르게 식별 및 해결 가능
3. 마지막으로 **[성능 최적화](./performance-optimization/)** 학습 —— 시스템 실행 효율 향상
:::

## 하위 페이지 탐색

### [문제 해결](./troubleshooting/)

일반적인 문제를 해결하는 포괄적인 문제 해결 가이드, 다음을 포함:
- Gateway 시작 실패
- 채널 연결 문제(WhatsApp, Telegram, Slack 등)
- 인증 오류 및 권한 문제
- 도구 호출 실패
- 세션 관리 예외

**사용 시나리오**: 구체적인 문제가 발생하여 빠르게 문제 해결 및 해결이 필요한 경우.

---

### [CLI 명령 참조](./cli-commands/)

완전한 명령 목록 및 사용법, 다음을 포함:
- Gateway 제어 명령(시작, 재시작, 상태)
- Agent 상호 작용 명령
- 모델 관리 및 구성
- 채널 관리(로그인, 상태, 로그아웃)
- 세션 관리(목록, 재설정, 삭제, 압축)
- 노드 관리(목록, 호출, 페어링)
- 스킬 관리 및 구성 마법사

**사용 시나리오**: 특정 명령의 사용법 및 매개변수를 빠르게 찾거나 CLI의 완전한 기능을 학습해야 하는 경우.

---

### [성능 최적화](./performance-optimization/)

Gateway 성능 및 응답 속도를 향상시키는 최적화 기술, 다음을 포함:
- 프롬프트 캐시(Prompt Caching)로 비용 절감
- 세션 프루닝(Session Pruning)으로 컨텍스트 감소
- 동시성 제어 및 리소스 관리
- 메모리 최적화 및 가비지 수집
- 로그 수준 튜닝

**사용 시나리오**: 시스템 실행이 느리거나 리소스 사용량이 높거나 사용자 경험을 향상시키고 싶을 때.
