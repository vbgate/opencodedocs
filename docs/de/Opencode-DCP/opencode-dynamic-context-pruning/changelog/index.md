# Änderungsprotokoll

In diesem Kapitel wird die Versionshistorie des DCP-Plugins dokumentiert, damit Sie die neuen Funktionen, Verbesserungen und Fixes jeder Version verstehen und Upgrade-Entscheidungen treffen können.

## Kapitelinhalt

| Seite | Beschreibung |
|--- | ---|
| [Versionshistorie](./version-history/) | Vollständige Aufzeichnung der Updates von v1.0.1 bis zur neuesten Version, einschließlich neuer Funktionen, Verbesserungen und Bugfixes |

## Leseempfehlung

Das Änderungsprotokoll eignet sich hauptsächlich für die folgenden Szenarien:

1. **Vor dem Upgrade** - Erfahren Sie, welche Änderungen die neue Version bringt, und bewerten Sie, ob ein Upgrade erforderlich ist
2. **Fehlerbehebung** - Bestätigen Sie, in welcher Version eine Funktion eingeführt oder ein Bug behoben wurde
3. **Entwicklungsrichtung verstehen** - Verstehen Sie durch die Versionshistorie die Entwicklung der Funktionen und die Optimierungsschwerpunkte von DCP

::: tip Aktuell bleiben
Es wird empfohlen, in der OpenCode-Konfiguration das Tag `@latest` zu verwenden, um automatisch die neuesten Funktionen und Fixes zu erhalten.
:::
