# Plattformfunktionen

Dieser Abschnitt stellt die beiden Hauptfunktionen von Plannotator vor: **Planüberprüfung** und **Codeüberprüfung**. Sie lernen, wie Sie KI-generierte Pläne visuell überprüfen, verschiedene Arten von Kommentaren hinzufügen, Bildmarkierungen anhängen und Git-Diff-Codeänderungen überprüfen können.

## Voraussetzungen

::: warning Bevor Sie beginnen
Stellen Sie sicher, dass Sie die folgenden Vorbereitungen abgeschlossen haben, bevor Sie mit diesem Abschnitt fortfahren:

- ✅ [Schnellstart](../start/getting-started/) Tutorial abgeschlossen
- ✅ Plannotator-Plugin installiert ([Claude Code](../start/installation-claude-code/) oder [OpenCode](../start/installation-opencode/))
:::

## Kapitelinhalt

### Planüberprüfung

Lernen Sie, wie Sie KI-generierte Ausführungspläne überprüfen, Änderungsvorschläge hinzufügen und die KI nach Ihren Absichten arbeiten lassen.

| Tutorial | Beschreibung |
|--- | ---|
| [Planüberprüfung-Grundlagen](./plan-review-basics/) | Lernen Sie, Plannotator zu verwenden, um KI-generierte Pläne visuell zu überprüfen, einschließlich Genehmigen oder Ablehnen von Plänen |
| [Plankommentare hinzufügen](./plan-review-annotations/) | Meistern Sie das Hinzufügen verschiedener Arten von Kommentaren zu Plänen (Löschen, Ersetzen, Einfügen, Kommentieren) |
| [Bildmarkierungen hinzufügen](./plan-review-images/) | Lernen Sie, Bilder an Planüberprüfungen anzuhängen und Stift-, Pfeil- und Kreistools für Markierungen zu verwenden |

### Codeüberprüfung

Lernen Sie, wie Sie Codeänderungen überprüfen, zeilenbasierte Kommentare hinzufügen und Probleme vor dem Committen erkennen.

| Tutorial | Beschreibung |
|--- | ---|
| [Codekommentare hinzufügen](./code-review-annotations/) | Meistern Sie das Hinzufügen zeilenbasierter Kommentare in Codeüberprüfungen (Kommentar/Vorschlag/Bedenken) |
| [Diff-Ansicht wechseln](./code-review-diff-types/) | Lernen Sie, zwischen verschiedenen Diff-Typen in Codeüberprüfungen zu wechseln (nicht committet/staged/letzter Commit/Branch) |

## Lernpfad

::: tip Empfohlene Lernreihenfolge
Wählen Sie den passenden Lernpfad basierend auf Ihrem Anwendungsszenario:

**Pfad A: Planüberprüfung priorisieren** (Empfohlen für Einsteiger)
1. [Planüberprüfung-Grundlagen](./plan-review-basics/) → Lernen Sie zuerst den grundlegenden Planüberprüfungsprozess
2. [Plankommentare hinzufügen](./plan-review-annotations/) → Lernen Sie, wie Sie Pläne präzise ändern
3. [Bildmarkierungen hinzufügen](./plan-review-images/) → Drücken Sie Ihre Absichten mit Bildern klarer aus
4. Danach lernen Sie die Codeüberprüfungsreihe

**Pfad B: Codeüberprüfung priorisieren** (Geeignet für Entwickler mit Codeüberprüfungserfahrung)
1. [Codeüberprüfung-Grundlagen](./code-review-basics/) → Machen Sie sich mit der Codeüberprüfungsoberfläche vertraut
2. [Codekommentare hinzufügen](./code-review-annotations/) → Lernen Sie zeilenbasierte Kommentare
3. [Diff-Ansicht wechseln](./code-review-diff-types/) → Meistern Sie verschiedene Diff-Typen
4. Danach lernen Sie die Planüberprüfungsreihe
:::
