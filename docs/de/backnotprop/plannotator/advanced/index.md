# Erweiterte Funktionen

Nachdem Sie die Grundlagen der Planprüfung und Codeprüfung beherrschen, führt Sie dieses Kapitel zu den erweiterten Fähigkeiten von Plannotator: Team-Zusammenarbeit durch Sharing, Integration mit Notiz-Apps, Remote-Entwicklungssupport sowie flexible Umgebungsvariablen-Konfiguration.

::: warning Voraussetzungen
Stellen Sie vor dem Studium dieses Kapitels sicher, dass Sie Folgendes abgeschlossen haben:
- [Schnellstart](../start/getting-started/): Grundlegende Konzepte von Plannotator verstehen
- [Grundlagen der Planprüfung](../platforms/plan-review-basics/): Grundlegende Planprüfung-Operationen beherrschen
:::

## Kapitelinhalt

| Kurs | Beschreibung | Geeignet für |
|--- | --- | ---|
| [URL-Sharing](./url-sharing/) | Freigabe von Plänen und Kommentaren per URL für backendlose Team-Zusammenarbeit | Prüfungsergebnisse mit Teammitgliedern teilen |
| [Obsidian-Integration](./obsidian-integration/) | Automatische Speicherung genehmigter Pläne im Obsidian-Vault | Wissensmanagement mit Obsidian |
| [Bear-Integration](./bear-integration/) | Speicherung von Plänen in Bear über x-callback-url | macOS/iOS-Benutzer mit Bear-Notizen |
| [Remote-Modus](./remote-mode/) | Nutzung in Remote-Umgebungen wie SSH, devcontainer, WSL | Remote-Entwicklungsszenarien |
| [Umgebungsvariablen](./environment-variables/) | Alle verfügbaren Umgebungsvariablen und deren Verwendung verstehen | Anpassung von Plannotator-Verhalten erforderlich |

## Empfohlener Lernpfad

```
┌─────────────────────────────────────────────────────────────────┐
│  Empfohlene Lernreihenfolge                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. URL-Sharing          ← Am häufigsten, für Team-Zusammenarbeit │
│       ↓                                                         │
│  2. Notiz-App-Integration ← Je nach verwendeter Notiz-App       │
│     ├─ Obsidian-Integration (Obsidian-Benutzer)                 │
│     └─ Bear-Integration (Bear-Benutzer)                        │
│       ↓                                                         │
│  3. Remote-Modus          ← Für Remote-Entwicklung              │
│       ↓                                                         │
│  4. Umgebungsvariablen   ← Bei Anpassungsbedarf                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

::: tip Nach Bedarf lernen
Sie müssen nicht alle Inhalte in der Reihenfolge absolvieren. Wenn Sie nur die Freigabefunktion benötigen, reicht URL-Sharing aus. Wenn Sie keine Remote-Entwicklung verwenden, können Sie den Remote-Modus überspringen.
:::
