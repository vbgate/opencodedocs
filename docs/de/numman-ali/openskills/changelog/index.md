# Änderungsprotokoll

Dieses Kapitel dokumentiert die Versionshistorie von OpenSkills und hilft Ihnen, die neuen Funktionen, Verbesserungen und Fehlerbehebungen jeder Version zu verstehen.

## Inhalt dieses Kapitels

| Seite | Beschreibung |
| --- | --- |
| [Änderungsprotokoll](./changelog/) | Vollständige Versionshistorie einschließlich neuer Funktionen, Verbesserungen, Fehlerbehebungen und Sicherheitsupdates |

## So verwenden Sie das Änderungsprotokoll

Das Änderungsprotokoll ist in umgekehrter chronologischer Reihenfolge der Versionsnummern organisiert (neueste Version zuerst). Jede Version enthält folgende Kategorien:

- **Neue Funktionen** - Neue hinzugefügte Funktionen und Befehle
- **Verbesserungen** - Optimierungen und Erweiterungen bestehender Funktionen
- **Fehlerbehebungen** - Behobene Bugs
- **Sicherheit** - Sicherheitsbezogene Verbesserungen

::: tip Upgrade-Empfehlung
Bevor Sie OpenSkills aktualisieren, empfiehlt es sich, das Änderungsprotokoll zu lesen, um die Versionsänderungen zu verstehen. Achten Sie besonders auf:
- **Breaking Changes** (Breaking Changes)
- Die Verwendung neuer Funktionen
- Die Status der Behobenen Probleme
:::
