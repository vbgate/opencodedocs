# Быстрый старт: от установки до первого сообщения

Добро пожаловать в Clawdbot! В этом разделе вы пройдёте полный путь от нуля до работающего AI-ассистента. Быстрый обзор возможностей или детальное изучение параметров — здесь найдётся всё необходимое.

---

## Обзор раздела

Этот раздел содержит 5 руководств, охватывающих весь процесс начала работы: от установки программного обеспечения и настройки AI-моделей и каналов связи до запуска демона Gateway, отправки первого сообщения и изучения встроенных механизмов безопасности. По завершении этого раздела у вас будет полноценный персональный AI-ассистент.

---

**Пояснения к порядку обучения**:

1. **Быстрый старт** (обязательно): базовая установка и настройка — основа для всего последующего обучения
2. **Мастер настройки** (рекомендуется): детальное изучение опций мастера для пользователей, желающих тонко настроить систему
3. **Запуск Gateway** (обязательно): запуск и управление демоном Gateway
4. **Первое сообщение** (обязательно): проверка правильности настройки и начало работы с AI-ассистентом
5. **DM Pairing и контроль доступа** (рекомендуется): изучение встроенных механизмов безопасности для защиты вашего AI-ассистента

::: tip Быстрый путь
Если вам нужно быстро попробовать Clawdbot, изучите только «Быстрый старт» и «Запуск Gateway», после чего сразу отправьте сообщение. Остальные руководства можно изучить позже по мере необходимости.
:::

## Требования

Перед началом изучения этого раздела убедитесь, что у вас есть:

- **Node.js**: ≥ 22.12.0 (проверьте командой `node -v`)
- **Операционная система**: macOS / Linux / Windows (WSL2)
- **Менеджер пакетов**: npm / pnpm / bun
- **Аккаунт AI-модели** (рекомендуется):
  - Аккаунт Anthropic Claude (подписка Pro/Max) с поддержкой OAuth-авторизации
  - Или API Key от провайдеров OpenAI / DeepSeek / OpenRouter

::: warning Внимание пользователям Windows
На **Windows настоятельно рекомендуется использовать WSL2**, так как:
- Многие каналы зависят от локальных бинарных файлов
- Демоны (launchd/systemd) недоступны в Windows
:::

## Навигация по подразделам

### [1. Быстрый старт](./getting-started/) ⭐ Основное руководство

**Что вы сможете делать после изучения**:
- ✅ Установить Clawdbot на ваше устройство
- ✅ Настроить аутентификацию AI-модели (Anthropic / OpenAI / другие провайдеры)
- ✅ Запустить демон Gateway
- ✅ Отправить первое сообщение через WebChat или настроенный канал

**Для кого**: все пользователи, обязательное к изучению

**Примерное время**: 15-20 минут

**Основное содержание**:
- Установка Clawdbot через npm/pnpm/bun
- Запуск мастера начальной настройки (onboarding)
- Запуск Gateway и проверка статуса
- Отправка тестового сообщения через CLI или канал

**Требования**: отсутствуют

### [2. Мастер настройки](./onboarding-wizard/)

**Что вы сможете делать после изучения**:
- ✅ Использовать интерактивный мастер для полной настройки
- ✅ Понимать разницу между режимами QuickStart и Manual
- ✅ Настраивать сеть Gateway, аутентификацию и Tailscale
- ✅ Настраивать провайдеров AI-моделей (setup-token и API Key)
- ✅ Включать каналы связи (WhatsApp, Telegram и др.)
- ✅ Устанавливать и управлять пакетами навыков

**Для кого**: пользователи, желающие тонко настроить систему и изучить расширенные опции

**Примерное время**: 20-30 минут

**Основное содержание**:
- Выбор между режимами QuickStart и Manual
- Настройка сети Gateway (порты, привязка, аутентификация)
- Способы аутентификации AI-модели (рекомендуется setup-token)
- Процесс настройки каналов связи
- Введение в систему навыков

**Требования**: [Быстрый старт](./getting-started/)

### [3. Запуск Gateway](./gateway-startup/) ⭐ Основное руководство

**Что вы сможете делать после изучения**:
- ✅ Запускать Gateway в foreground-режиме через командную строку
- ✅ Настраивать Gateway как фоновый демон (macOS LaunchAgent / Linux systemd / Windows Scheduled Task)
- ✅ Понимать различия между режимами привязки (loopback / LAN / Tailnet) и способами аутентификации
- ✅ Переключаться между режимами разработки и продакшена
- ✅ Использовать `--force` для принудительного освобождения занятого порта

**Для кого**: все пользователи, обязательное к изучению

**Примерное время**: 15-20 минут

**Основное содержание**:
- Режим foreground vs демон
- Выбор режима привязки (loopback / LAN / Tailnet / Auto)
- Настройка аутентификации (Token / Password / Tailscale Identity)
- Режим разработки (`--dev`) и продакшен
- Команды управления сервисом (install / start / stop / restart)
- Разрешение конфликтов портов (`--force`)

**Требования**: [Мастер настройки](./onboarding-wizard/)

## Часто задаваемые вопросы

### В: Нужно ли изучать все руководства?

**О**: Не обязательно. Если вам нужно быстро начать работу, достаточно изучить два основных руководства — «Быстрый старт» и «Запуск Gateway», после чего можно сразу использовать Clawdbot. Остальные руководства можно изучать по мере необходимости.

### В: Будет ли проблемой, если я пропущу некоторые руководства?

**О**: Нет. Каждое руководство самодостаточно, но «Быстрый старт» является основой, содержащей установку и базовую настройку, поэтому рекомендуется начать с него. Другие руководства можно изучать выборочно в зависимости от ваших потребностей.

### В: Я уже знаком с AI-ассистентами, могу ли я пропустить базовые руководства?

**О**: Да. Если вы уже знакомы с подобными инструментами AI-ассистентов, вы можете пропустить «Быстрый старт» и сразу перейти к «Мастеру настройки» и «Запуску Gateway», чтобы узнать о специфических параметрах и способах запуска Clawdbot.

### В: Что я смогу делать после изучения этого раздела?

**О**: После завершения этого раздела у вас будет работающая система Clawdbot, позволяющая:
- Общаться с AI-ассистентом через WebChat или несколько каналов
- Поручать AI выполнение базовых задач (поиск информации, генерация кода, суммирование текста и т.д.)
- Использовать механизм DM Pairing для обеспечения безопасности
- Продолжить изучение расширенных функций (многоканальная настройка, система инструментов, платформа навыков и т.д.)
