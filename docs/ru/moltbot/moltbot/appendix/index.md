# Приложение

В этой главе представлены расширенная документация и ресурсы для разработки Clawdbot, включая полный справочник по конфигурации, спецификацию протокола Gateway WebSocket API, варианты развертывания и руководство по разработке.

::: info Сценарии использования
Эта глава предназначена для пользователей, которым необходимо глубоко понять внутренние механизмы Clawdbot, выполнять расширенную конфигурацию, развертывание или участвовать в разработке. Если вы только начинаете использовать, рекомендуется сначала пройти главу [Быстрый старт](../../start/getting-started/).
:::

## Навигация по подстраницам

### [Полный справочник по конфигурации](./config-reference/)
**Подробный справочник по конфигурации** — охватывает все параметры конфигурации, значения по умолчанию и примеры. Найдите полную документацию по модулям Gateway, Agent, каналов, инструментов и других.

### [Протокол Gateway WebSocket API](./api-protocol/)
**Документация по спецификации протокола** — полная спецификация протокола Gateway WebSocket, включая определения конечных точек, форматы сообщений, методы аутентификации и механизмы подписки на события. Подходит для разработчиков, которым необходимо создавать собственные клиенты или интегрировать Gateway.

### [Варианты развертывания](./deployment/)
**Руководство по способам развертывания** — способы развертывания на различных платформах: локальная установка, Docker, VPS, Fly.io, Nix и другие. Узнайте, как запускать Clawdbot в различных средах.

### [Руководство по разработке](./development/)
**Документация для разработчиков** — сборка из исходного кода, разработка плагинов, тестирование и процесс внесения вклада. Узнайте, как участвовать в разработке проекта Clawdbot или писать собственные плагины.

## Рекомендации по пути обучения

В зависимости от ваших потребностей выберите подходящий путь обучения:

### Инженеры по конфигурации и эксплуатации
1. Сначала прочитайте [Полный справочник по конфигурации](./config-reference/) — узнайте все настраиваемые параметры
2. Ознакомьтесь с [Вариантами развертывания](./deployment/) — выберите подходящую схему развертывания
3. При необходимости обратитесь к документации Gateway WebSocket API для интеграции

### Разработчики приложений
1. Прочитайте [Протокол Gateway WebSocket API](./api-protocol/) — поймите механизм протокола
2. Ознакомьтесь с [Полным справочником по конфигурации](./config-reference/) — узнайте, как настраивать связанные функции
3. Обращайтесь к примерам протокола для создания клиентов

### Разработчики плагинов/функций
1. Прочитайте [Руководство по разработке](./development/) — узнайте среду разработки и процесс сборки
2. Углубитесь в [Протокол Gateway WebSocket API](./api-protocol/) — поймите архитектуру Gateway
3. Обратитесь к [Полному справочнику по конфигурации](./config-reference/) — узнайте систему конфигурации

## Предварительные требования

::: warning Предварительные знания
Перед углублением в эту главу рекомендуется завершить следующее:
- ✅ Завершили [Быстрый старт](../../start/getting-started/)
- ✅ Настроили как минимум один канал (например, [WhatsApp](../../platforms/whatsapp/) или [Telegram](../../platforms/telegram/))
- ✅ Поняли базовую конфигурацию моделей ИИ (см. [Модели ИИ и аутентификация](../../advanced/models-auth/))
- ✅ Имеете базовое понимание JSON-конфигурационных файлов и TypeScript
:::
