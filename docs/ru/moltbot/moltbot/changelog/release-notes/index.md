# История версий: новые функции, улучшения и критические изменения

Clawdbot постоянно развивается, и каждый выпуск приносит новые функции, улучшения производительности и усиление безопасности. Этот журнал поможет вам быстро узнать об эволюции версий и решить, когда обновляться и на что обращать внимание при обновлении.

## Чему вы научитесь

- Узнаете о новых функциях и ключевых изменениях последних версий
- Поймёте критические изменения каждой версии, чтобы избежать сбоев при обновлении
- Просмотрите список исправлений, чтобы убедиться, что ваша проблема решена
- Отслеживайте эволюцию функций и планируйте использование новых возможностей

::: tip О формате номеров версий
Формат номера версии: `ГГГГ.М.Д` (год.месяц.день)

- **Основная версия**: изменение года или месяца обычно означает значительное обновление функций
- **Патч-версия**: `-1`, `-2`, `-3` обозначают исправительные версии, содержащие только исправления ошибок
:::

## Рекомендации по обновлению

### Проверки перед обновлением

Перед обновлением до новой версии рекомендуется:

1. **Прочитать критические изменения**: проверьте, есть ли критические изменения, влияющие на вашу конфигурацию
2. **Создать резервную копию конфигурации**: сделайте резервную копию `~/.clawdbot/clawdbot.json`
3. **Запустить диагностику**: выполните `clawdbot doctor`, чтобы убедиться в работоспособности текущей системы
4. **Проверить зависимости**: убедитесь, что версия Node.js соответствует требованиям (≥22)

### Проверка после обновления

После завершения обновления выполните следующие проверки:

```bash
# 1. Проверка версии
clawdbot --version

# 2. Проверка статуса
clawdbot status

# 3. Проверка подключения каналов
clawdbot channels status

# 4. Тест отправки сообщения
clawdbot message "Hello" --target=<your-channel>
```
