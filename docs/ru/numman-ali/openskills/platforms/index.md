# Функции платформы

В этом разделе подробно рассматриваются основные команды и функции управления навыками OpenSkills CLI, помогая вам перейти от базового использования к мастерству.

## Предварительные требования

::: warning Перед началом убедитесь
Прежде чем приступить к изучению этого раздела, убедитесь, что вы завершили раздел [Быстрый старт](../start/), в частности:
- [Установка OpenSkills](../start/installation/): OpenSkills CLI успешно установлен
- [Установка первого навыка](../start/first-skill/): Вы понимаете базовый процесс установки навыков
:::

## Содержание раздела

Этот раздел включает 6 тем, охватывающих все основные функции OpenSkills:

### [Подробное описание команд](./cli-commands/)

Полное описание всех 7 команд, параметров и флагов с краткой справочной таблицей. Подходит для пользователей, которым нужно быстро найти информацию об использовании команд.

### [Источники установки](./install-sources/)

Подробное описание трёх источников установки: репозитории GitHub, локальные пути, приватные Git-репозитории. Сценарии применения, формат команд и важные замечания для каждого способа.

### [Глобальная установка vs установка в проект](./global-vs-project/)

Объяснение различий между установкой с флагом `--global` и установкой по умолчанию (в проект), помогающее выбрать подходящее место установки и понять правила приоритета поиска навыков.

### [Просмотр установленных навыков](./list-skills/)

Научитесь использовать команду `list` для просмотра установленных навыков и понимать значение меток расположения `(project)` и `(global)`.

### [Обновление навыков](./update-skills/)

Руководство по использованию команды `update` для обновления установленных навыков с поддержкой обновления всех навыков или выборочного обновления для синхронизации с исходным репозиторием.

### [Удаление навыков](./remove-skills/)

Описание двух способов удаления: интерактивная команда `manage` и скриптовая команда `remove` для эффективного управления библиотекой навыков.

Выберите подходящий путь обучения в зависимости от ваших потребностей:


Переходите напрямую к нужной теме в зависимости от текущей задачи:
