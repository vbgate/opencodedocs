## Вопросы о ключевых концепциях

### В чем разница между OpenSkills и Claude Code?

**Краткий ответ**: OpenSkills — это "универсальный установщик", Claude Code — это "официальный агент".

**Подробное объяснение**:

| Сравнение | OpenSkills | Claude Code |
|--- | --- | ---|
| **Назначение** | Универсальный загрузчик навыков | Официальный AI-агент для кодирования от Anthropic |
| **Поддержка** | Все AI-агенты (Cursor, Windsurf, Aider и др.) | Только Claude Code |
| **Формат навыков** | Полностью совместим с Claude Code (`SKILL.md`) | Официальная спецификация |
| **Способ установки** | Из GitHub, локального пути, приватного репозитория | Из встроенного Marketplace |
| **Хранилище навыков** | `.claude/skills/` или `.agent/skills/` | `.claude/skills/` |
| **Способ вызова** | `npx openskills read <name>` | Встроенный инструмент `Skill()` |

**Ключевая ценность**: OpenSkills позволяет другим агентам использовать систему навыков Anthropic без необходимости ожидания отдельной реализации каждым агентом.

### Почему CLI, а не MCP?

**Основная причина**: Навыки — это статические файлы, MCP — это динамические инструменты, они решают разные задачи.

| Сравнение | MCP (Model Context Protocol) | OpenSkills (CLI) |
|--- | --- | ---|
| **Сценарий использования** | Динамические инструменты, вызовы API в реальном времени | Статические инструкции, документация, скрипты |
| **Требования к запуску** | Требуется MCP-сервер | Не нужен сервер (чистые файлы) |
| **Поддержка агентов** | Только агенты с поддержкой MCP | Все агенты, которые могут читать `AGENTS.md` |
| **Сложность** | Требуется развертывание сервера | Нулевая конфигурация |

**Ключевые моменты**:

- **Навыки — это файлы**: SKILL.md — это статические инструкции + ресурсы (references/, scripts/, assets/), сервер не нужен
- **Не требуется поддержка агентов**: Любой агент, который может выполнять команды shell, может использовать это
- **Соответствует официальному дизайну**: Система навыков Anthropic изначально спроектирована как файловая система, а не как MCP

**Итог**: MCP и система навыков решают разные задачи. OpenSkills сохраняет легкость и универсальность навыков, не требуя, чтобы каждый агент поддерживал MCP.

## Вопросы об установке и конфигурации

### Что делать при сбое установки?

**Распространенные ошибки и решения**:

#### Ошибка 1: Ошибка клонирования

```bash
Error: Git clone failed
```

**Возможные причины**:
- Проблемы с сетью (невозможно получить доступ к GitHub)
- Git не установлен или версия устарела
- Для приватного репозитория не настроен SSH-ключ

**Решения**:

1. Проверьте, установлен ли Git:
   ```bash
   git --version
   # Должно показать: git version 2.x.x
   ```

2. Проверьте сетевое подключение:
   ```bash
   # Проверьте возможность доступа к GitHub
   ping github.com
   ```

3. Для приватного репозитория настройте SSH:
   ```bash
   # Проверьте SSH-подключение
   ssh -T git@github.com
   ```

#### Ошибка 2: Путь не существует

```bash
Error: Path does not exist: ./nonexistent-path
```

**Решения**:
- Убедитесь в правильности локального пути
- Используйте абсолютный или относительный путь:
  ```bash
  # Абсолютный путь
  npx openskills install /Users/dev/my-skills

  # Относительный путь
  npx openskills install ./my-skills
  ```

#### Ошибка 3: SKILL.md не найден

```bash
Error: No valid SKILL.md found
```

**Решения**:

1. Проверьте структуру каталога навыка:
   ```bash
   ls -la ./my-skill
   # Должен содержать SKILL.md
   ```

2. Убедитесь, что SKILL.md имеет корректный YAML frontmatter:
   ```markdown
   ---
   name: my-skill
   description: Skill description
   ---

   # Skill content
   ```

### В какой каталог устанавливаются навыки?

**Место установки по умолчанию** (локально для проекта):
```bash
.claude/skills/
```

**Место глобальной установки** (с использованием `--global`):
```bash
~/.claude/skills/
```

**Режим Universal** (с использованием `--universal`):
```bash
.agent/skills/
```

**Приоритет поиска навыков** (от высшего к низшему):
1. `./.agent/skills/` (локально для проекта, Universal)
2. `~/.agent/skills/` (глобально, Universal)
3. `./.claude/skills/` (локально для проекта, по умолчанию)
4. `~/.claude/skills/` (глобально, по умолчанию)

**Просмотр местоположения установленных навыков**:
```bash
npx openskills list
# Вывод показывает метки [project] или [global]
```

### Как сосуществовать с Claude Code Marketplace?

**Проблема**: Хочу использовать и Claude Code, и OpenSkills, как избежать конфликтов?

**Решение**: Используйте режим Universal

```bash
# Установка в .agent/skills/ вместо .claude/skills/
npx openskills install anthropics/skills --universal
```

**Почему это работает**:

| Каталог | Кто использует | Описание |
|--- | --- | ---|
| `.claude/skills/` | Claude Code | Используется Claude Code Marketplace |
| `.agent/skills/` | OpenSkills | Используется другими агентами (Cursor, Windsurf) |

**Предупреждение о конфликте**:

При установке из официального репозитория OpenSkills покажет:
```
⚠️  Warning: These skills are also available in Claude Code Marketplace.
   Installing to .claude/skills/ may cause conflicts.
   Use --universal to install to .agent/skills/ instead.
```

## Вопросы об использовании

### Навык не появляется в AGENTS.md?

**Симптом**: После установки навыка он не отображается в AGENTS.md.

**Шаги диагностики**:

#### 1. Убедитесь, что синхронизация выполнена

После установки навыка необходимо выполнить команду `sync`:

```bash
npx openskills install anthropics/skills
# Выберите навыки...

# Обязательно выполните sync!
npx openskills sync
```

#### 2. Проверьте местоположение AGENTS.md

```bash
# По умолчанию AGENTS.md находится в корневой директории проекта
cat AGENTS.md
```

Если используется пользовательский путь вывода, убедитесь, что путь правильный:
```bash
npx openskills sync -o custom-path/AGENTS.md
```

#### 3. Проверьте, выбран ли навык

Команда `sync` является интерактивной, убедитесь, что вы выбрали навыки для синхронизации:

```bash
npx openskills sync

? Select skills to sync:
  ◉ pdf                  [выбран]
  ◯ check-branch-first   [не выбран]
```

#### 4. Проверьте содержимое AGENTS.md

Убедитесь в правильности XML-тегов:

```xml
<available_skills>

<skill>
<name>pdf</name>
<description>Comprehensive PDF manipulation toolkit...</description>
<location>project</location>
</skill>

</available_skills>
```

### Как обновить навыки?

**Обновить все навыки**:
```bash
npx openskills update
```

**Обновить указанные навыки** (через запятую):
```bash
npx openskills update pdf,git-workflow
```

**Распространенные вопросы**:

#### Навык не обновлен

**Симптом**: После выполнения `update` показывается "skipped"

**Причина**: При установке навыка не была записана информация об источнике (поведение старой версии)

**Решение**:
```bash
# Переустановите для записи источника
npx openskills install anthropics/skills
```

#### Локальные навыки нельзя обновить

**Симптом**: При установке из локального пути, при update возникает ошибка

**Причина**: Навыки из локального пути нужно обновлять вручную

**Решение**:
```bash
# Переустановите из локального пути
npx openskills install ./my-skill
```

### Как удалить навыки?

**Метод 1: Интерактивное удаление**

```bash
npx openskills manage
```

Выберите навык для удаления, нажмите пробел для выбора, Enter для подтверждения.

**Метод 2: Прямое удаление**

```bash
npx openskills remove <skill-name>
```

**После удаления**: Не забудьте выполнить `sync` для обновления AGENTS.md:
```bash
npx openskills sync
```

**Распространенные вопросы**:

#### Случайное удаление навыка

**Способ восстановления**:
```bash
# Переустановите из источника
npx openskills install anthropics/skills
# Выберите случайно удаленный навык
```

#### После удаления навык все еще отображается в AGENTS.md

**Решение**: Повторная синхронизация
```bash
npx openskills sync
```

## Продвинутые вопросы

### Как навыки распределяются между проектами?

**Сценарий**: Несколько проектов используют один и тот же набор навыков, не хочу устанавливать их многократно.

**Решение 1: Глобальная установка**

```bash
# Установите один раз глобально
npx openskills install anthropics/skills --global

# Все проекты могут использовать
cd project-a
npx openskills read pdf

cd project-b
npx openskills read pdf
```

**Преимущества**:
- Установите один раз, используйте везде
- Уменьшение использования диска

**Недостатки**:
- Навыки не в проекте, не включаются в контроль версий

**Решение 2: Символические ссылки**

```bash
# 1. Глобальная установка навыка
npx openskills install anthropics/skills --global

# 2. Создание символической ссылки в проекте
cd project-a
ln -s ~/.claude/skills/pdf .claude/skills/pdf

# 3. При sync будет распознано как местоположение [project]
npx openskills sync
```

**Преимущества**:
- Навык в проекте (метка `[project]`)
- Контроль версий может включать символические ссылки
- Установите один раз, используйте в нескольких местах

**Недостатки**:
- Символические ссылки на некоторых системах требуют прав доступа

**Решение 3: Git Submodule**

```bash
# Добавьте репозиторий навыков как submodule в проект
cd project-a
git submodule add https://github.com/anthropics/skills.git .claude/skills-repo

# Установка навыков из submodule
npx openskills install .claude/skills-repo/pdf
```

**Преимущества**:
- Полный контроль версий
- Можно указать версию навыка

**Недостатки**:
- Сложная конфигурация

### Символическая ссылка недоступна?

**Симптом**:

```bash
ln -s ~/dev/my-skills/my-skill .claude/skills/my-skill
# ln: failed to create symbolic link: Operation not permitted
```

**Решения по системам**:

#### macOS

1. Откройте "Системные настройки"
2. Перейдите в "Безопасность и конфиденциальность"
3. В "Полный доступ к диску" разрешите вашему терминалу

#### Windows

Windows изначально не поддерживает символические ссылки, рекомендуется:
- **Использовать Git Bash**: встроенная поддержка символических ссылок
- **Использовать WSL**: подсистема Linux поддерживает символические ссылки
- **Включить режим разработчика**: Параметры → Обновление и безопасность → Режим разработчика

```bash
# Создание символической ссылки в Git Bash
ln -s /c/dev/my-skills/my-skill .claude/skills/my-skill
```

#### Linux

Проверьте права доступа к файловой системе:

```bash
# Проверьте права доступа к каталогу
ls -la .claude/

# Добавьте права на запись
chmod +w .claude/
```

### Навык не найден?

**Симптом**:

```bash
npx openskills read my-skill
# Error: Skill not found: my-skill
```

**Шаги диагностики**:

#### 1. Убедитесь, что навык установлен

```bash
npx openskills list
```

#### 2. Проверьте регистр названия навыка

```bash
# ❌ Ошибка (заглавные буквы)
npx openskills read My-Skill

# ✅ Правильно (строчные буквы)
npx openskills read my-skill
```

#### 3. Проверьте, не перекрыт ли навык навыком с более высоким приоритетом

```bash
# Просмотрите все местоположения навыков
ls -la .claude/skills/my-skill
ls -la ~/.claude/skills/my-skill
ls -la .agent/skills/my-skill
ls -la ~/.agent/skills/my-skill
```

**Правило поиска навыков**: Местоположение с наивысшим приоритетом перекрывает одноименные навыки из других местоположений.
