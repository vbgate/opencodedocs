# Расширенные функции

Эта глава глубоко объясняет низкоуровневые механизмы и расширенные параметры конфигурации Supermemory. Вы научитесь оптимизировать производительность движка памяти и настраивать поведение плагина в соответствии с требованиями проекта.

## Содержание главы

<div class="grid-cards">

### [Принцип упреждающего сжатия](./compaction/)

Глубокое понимание механизма автоматического сжатия, триггерного по порогу токенов. Узнайте, как Supermemory перед переполнением контекста активно генерирует структурированную сводку и сохраняет ее постоянно, предотвращая проблему "потери памяти" в длительных сессиях.

### [Подробная настройка](./configuration/)

Настройте порог сжатия, правила триггеризации по ключевым словам и параметры поиска. Овладей всеми опциями файла конфигурации, чтобы Supermemory идеально соответствовал вашему рабочему процессу.

</div>

## Путь обучения

```
┌─────────────────────────────────────────────────────────────┐
│  Шаг 1              Шаг 2                                    │
│  Принцип упреждающего   →   Подробная настройка              │
│  сжатия              (Понимание механизма)      (Настройка) │
└─────────────────────────────────────────────────────────────┘
```

**Рекомендуемый порядок**:

1. **Сначала изучите принцип сжатия**: Поймите, как Supermemory автоматически управляет контекстом — это основа расширенной настройки.
2. **Затем изучите подробную настройку**: Только после понимания механизма вы сможете принимать обоснованные решения о конфигурации (например, какой порог подходит).
