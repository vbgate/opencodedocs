# Быстрый старт

Этот раздел поможет вам начать работу с Plannotator с нуля. Вы узнаете, что такое Plannotator, какие проблемы он решает, и завершите установку и настройку в используемом вами AI Coding Agent (Claude Code или OpenCode).

## Содержание раздела

<div class="grid-cards">

### [Быстрый старт](./getting-started/)

Узнайте основные функции и сценарии использования Plannotator, завершите первое знакомство за 5 минут. Подходит для всех новых пользователей в качестве первого урока.

### [Установка плагина Claude Code](./installation-claude-code/)

Установите и настройте плагин Plannotator в Claude Code. Поддерживает два способа: систему плагинов и ручные Hook, включая локальную и удалённую конфигурацию.

### [Установка плагина OpenCode](./installation-opencode/)

Установите и настройте плагин Plannotator в OpenCode, активируйте tool `submit_plan` и команду `/plannotator-review`.

</div>

## Путь обучения

::: tip Рекомендуемый порядок
1. **Сначала прочитайте [Быстрый старт](./getting-started/)**: Узнайте, что такое Plannotator, что он может делать, сформируйте общее понимание
2. **Затем выберите руководство по установке**: Выберите соответствующее руководство по установке в зависимости от используемого вами AI Coding Agent
   - Используете Claude Code → [Установка плагина Claude Code](./installation-claude-code/)
   - Используете OpenCode → [Установка плагина OpenCode](./installation-opencode/)
:::

После завершения установки вы можете начать использовать Plannotator для проверки планов и кода.

## Следующие шаги

После завершения этого раздела продолжайте изучать **[Функции платформы](../platforms/)**, чтобы освоить полное использование проверки планов и кода:

- Основы проверки планов: одобрение, отклонение, добавление комментариев
- Аннотирование изображений: прикрепление снимков экрана и аннотирование в планах
- Проверка кода: визуализация Git diff, добавление комментариев на уровне строк
