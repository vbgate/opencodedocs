# Функции платформы

В этой главе представлены две ключевые функции Plannotator: **обзор планов** и **обзор кода**. Вы научитесь визуально просматривать планы, созданные ИИ, добавлять различные типы аннотаций, прикреплять изображения с разметкой и проверять изменения кода через git diff.

## Предварительные требования

::: warning Перед началом убедитесь
Прежде чем приступить к изучению этой главы, убедитесь, что вы выполнили следующие подготовительные шаги:

- ✅ Завершили обучение по [быстрому старту](../start/getting-started/)
- ✅ Установили плагин Plannotator ([Claude Code](../start/installation-claude-code/) или [OpenCode](../start/installation-opencode/))
:::

## Содержание главы

### Обзор планов

Узнайте, как проверять планы выполнения, созданные ИИ, добавлять предложения по изменениям и заставить ИИ действовать в соответствии с вашими намерениями.

| Учебный материал | Описание |
| --- | --- |
| [Основы обзора планов](./plan-review-basics/) | Научитесь визуально просматривать планы, созданные ИИ, включая утверждение или отклонение планов |
| [Добавление аннотаций к планам](./plan-review-annotations/) | Освойте добавление различных типов аннотаций в планы (удаление, замена, вставка, комментарий) |
| [Добавление разметки изображений](./plan-review-images/) | Научитесь прикреплять изображения к обзору планов и использовать инструменты рисования, стрелки и круги для разметки |

### Обзор кода

Узнайте, как проверять изменения кода, добавлять построчные комментарии и находить проблемы перед фиксацией.

| Учебный материал | Описание |
| --- | --- |
| [Добавление комментариев к коду](./code-review-annotations/) | Освойте добавление построчных комментариев при обзоре кода (comment/suggestion/concern) |
| [Переключение типов diff](./code-review-diff-types/) | Научитесь переключаться между различными типами diff (uncommitted/staged/last commit/branch) в обзоре кода |

## Путь обучения

::: tip Рекомендуемый порядок обучения
В зависимости от вашего сценария использования, выберите подходящий путь обучения:

**Путь A: Приоритет обзора планов** (рекомендуется для начинающих)
1. [Основы обзора планов](./plan-review-basics/) → Сначала изучите базовый процесс обзора планов
2. [Добавление аннотаций к планам](./plan-review-annotations/) → Научитесь точно изменять планы
3. [Добавление разметки изображений](./plan-review-images/) → Используйте изображения для более четкого выражения намерений
4. Затем изучите серию обзора кода

**Путь B: Приоритет обзора кода** (для разработчиков с опытом code review)
1. [Основы обзора кода](./code-review-basics/) → Ознакомьтесь с интерфейсом обзора кода
2. [Добавление комментариев к коду](./code-review-annotations/) → Изучите построчные комментарии
3. [Переключение типов diff](./code-review-diff-types/) → Освойте различные типы diff
4. Затем изучите серию обзора планов
:::
