# 快速开始

本章节帮助你从零开始使用 Plannotator。你将了解 Plannotator 是什么、它能解决什么问题，并在你使用的 AI Coding Agent（Claude Code 或 OpenCode）中完成安装配置。

## 本章内容

<div class="grid-cards">

### [快速开始](./getting-started/)

了解 Plannotator 的核心功能和使用场景，在 5 分钟内完成首次体验。适合所有新用户作为入门第一课。

### [安装 Claude Code 插件](./installation-claude-code/)

在 Claude Code 中安装并配置 Plannotator 插件，支持插件系统和手动 Hook 两种方式，涵盖本地和远程环境配置。

### [安装 OpenCode 插件](./installation-opencode/)

在 OpenCode 中安装并配置 Plannotator 插件，启用 `submit_plan` tool 和 `/plannotator-review` 命令。

</div>

## 学习路径

::: tip 推荐顺序
1. **先读 [快速开始](./getting-started/)**：了解 Plannotator 是什么、能做什么，建立整体认知
2. **再选一个安装教程**：根据你使用的 AI Coding Agent 选择对应的安装指南
   - 使用 Claude Code → [安装 Claude Code 插件](./installation-claude-code/)
   - 使用 OpenCode → [安装 OpenCode 插件](./installation-opencode/)
:::

完成安装后，你就可以开始使用 Plannotator 进行计划评审和代码评审了。

## 下一步

完成本章节后，继续学习 **[平台功能](../platforms/)**，掌握计划评审和代码评审的完整用法：

- 计划评审基础：批准、拒绝、添加注释
- 图像标注：在计划中附加截图并标注
- 代码评审：可视化 Git diff，添加行级注释
