# 扩展 Superpowers

本章节教你如何扩展 Superpowers：编写自定义技能、测试技能以及向项目贡献。学完后，你可以打造专属的 AI 编码工作流，并与社区分享你的成果。

## 子页面导航

### [编写自定义技能](../writing-skills/)

学会遵循最佳实践编写自己的技能。从技能结构、元数据定义到实际实现，完整掌握技能开发的每个环节。

### [测试技能](../testing-skills/)

使用子代理测试自己编写的技能，确保技能在各种场景下都能正确工作。学习编写测试用例和验证方法。

### [贡献指南](../contributing/)

向 Superpowers 项目贡献技能和改进。了解贡献流程、代码规范和提交流程，让更多人受益于你的工作。

## 学习路径建议

我们推荐按以下顺序学习本章节内容：

1. **编写自定义技能**：先掌握技能的基本结构和开发规范
2. **测试技能**：确保你编写的技能质量可靠
3. **贡献指南**：将优秀的技能分享给社区

::: tip 小贴士
在学习编写技能前，建议你先完成 [如何使用技能](../../start/using-skills/) 和至少 2-3 个核心工作流的学习，这样能更好地理解技能的应用场景。
:::

## 前置条件

开始学习本章节前，请确保：

- ✅ 已完成 [快速开始](../../start/quick-start/)
- ✅ 已熟悉至少 2 个核心工作流（如 [设计工作流](../../core-workflows/design-workflow/) 和 [TDD 工作流](../../core-workflows/tdd-workflow/)）
- ✅ 了解基本的 Markdown 写作规范
- ✅ 有一定的编程经验（JavaScript/TypeScript 优先）
