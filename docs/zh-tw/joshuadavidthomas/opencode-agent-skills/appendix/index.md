# 附錄

本章節提供 OpenCode Agent Skills 的參考資料和開發規範，幫助你編寫高品質的技能並快速查閱 API 介面。

## 本章內容

| 頁面 | 說明 |
| --- | --- |
| [技能開發最佳實踐](./best-practices/) | 掌握編寫高品質技能的技巧和規範，包括命名規範、描述編寫、目錄組織、腳本使用等 |
| [API 工具參考](./api-reference/) | 查看所有可用工具的詳細參數和返回值，包括 `get_available_skills`、`read_skill_file`、`run_skill_script`、`use_skill` |

## 學習路徑建議

::: tip 推薦順序
1. **先學最佳實踐** → 了解技能開發的規範和技巧，避免常見錯誤
2. **再查 API 參考** → 在實際開發中遇到問題時，查閱具體的參數和返回值
:::

這兩篇內容的定位不同：

- **最佳實踐**：適合在開始編寫技能前通讀一遍，建立正確的開發習慣
- **API 參考**：適合作為工具書，在需要時查閱具體細節

## 前置條件

- 了解 [技能發現機制](../platforms/skill-discovery-mechanism/)，理解插件如何找到技能
:::
