# 進階功能

本章節深入講解 DCP 的進階特性，幫助你理解外掛的內部機制，並在複雜場景下正確使用 DCP。

::: warning 前置條件
學習本章節前，請確保你已完成：
- [安裝與快速開始](../start/getting-started/) - 了解 DCP 的基本安裝和使用
- [設定全解](../start/configuration/) - 熟悉 DCP 的設定系統
- [自動修剪策略詳解](../platforms/auto-pruning/) - 理解 DCP 的核心修剪策略
:::

| [Prompt 快取影響](./prompt-caching/) | DCP 修剪對 Prompt Caching 的影響 | 優化快取命中率 |
| [子代理處理](./subagent-handling/) | DCP 在子代理會話中的行為和限制 | 使用 Task 工具時 |

## 學習路徑建議

```
保護機制 → 狀態持久化 → Prompt 快取影響 → 子代理處理
   ↓           ↓              ↓              ↓
 必學       按需學習       效能優化時學     用子代理時學
```

**推薦順序**：

1. **保護機制**（必學）- 這是最重要的進階特性，理解它能避免 DCP 誤刪關鍵內容
2. **狀態持久化**（按需）- 如果你想了解 DCP 如何記錄統計資料，或需要除錯狀態問題
3. **Prompt 快取影響**（效能優化時）- 當你關注 API 成本優化時，需要權衡修剪和快取的關係
4. **子代理處理**（用子代理時）- 如果你使用 OpenCode 的 Task 工具派發子任務，需要了解 DCP 的限制
