# よくある質問とトラブルシューティング

この章では、oh-my-opencode の使用中に遭遇する一般的な問題を解決する方法を説明します。設定診断から使用のコツ、セキュリティの推奨事項まで、問題を迅速に特定して解決策を見つけることができます。

## 学習パス

この順序に従って、問題のトラブルシューティングとベストプラクティスを段階的に習得してください：

### 1. [設定診断とトラブルシューティング](./troubleshooting/)

Doctor コマンドを使用して、設定の問題を迅速に診断し解決する方法を学びます。
- Doctor コマンドを実行して完全なヘルスチェックを行います
- 17+ のチェック結果を解釈します（インストール、設定、認証、依存関係、ツール、更新）
- 一般的な設定の問題を特定して修正します
- 詳細モードと JSON 出力を使用して高度な診断を行います

### 2. [よくある質問](./faq/)

使用中の一般的な問題を特定して解決します。
- インストールと設定の問題の迅速な回答
- 使用のコツとベストプラクティス（ultrawork、プロキシ呼び出し、バックグラウンドタスク）
- Claude Code 互換性の注意事項
- セキュリティの警告とパフォーマンス最適化の推奨事項
- 貢献とヘルプのリソース

## 前提条件

この章を開始する前に、以下を確認してください：
- [クイックインストールと設定](../start/installation/) を完了していること
- oh-my-opencode の基本的な設定ファイル構造に慣れていること
- 特定の問題に遭遇しているか、ベストプラクティスを理解したいこと

::: tip 推奨読書タイミング
基本的なインストールを完了した後、まずよくある質問（FAQ）を一通り読んで、一般的な落とし穴とベストプラクティスを理解することをお勧めします。特定の問題に遭遇したときは、トラブルシューティングツールを使用して診断を行います。
:::

## クイックトラブルシューティングガイド

緊急の問題に遭遇した場合は、以下の手順で迅速にトラブルシューティングを行います：

```bash
# ステップ 1: 完全な診断を実行
bunx oh-my-opencode doctor

# ステップ 2: 詳細なエラー情報を表示
bunx oh-my-opencode doctor --verbose

# ステップ 3: 特定のカテゴリを確認（例：認証）
bunx oh-my-opencode doctor --category authentication

# ステップ 4: まだ解決しない場合は、よくある質問を確認するか
# GitHub Issues でヘルプを求めてください
```
