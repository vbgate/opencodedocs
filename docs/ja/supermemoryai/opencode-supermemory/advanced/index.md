# 高度な機能

この章では Supermemory の底層メカニズムと高度な設定オプションを深く説明します。記憶エンジンのパフォーマンスを最適化し、プロジェクトの要件に応じてプラグインの動作をカスタマイズする方法を学びます。

## 本章の内容

<div class="grid-cards">

### [先制圧縮の原理](./compaction/)

トークンしきい値でトリガーされる自動圧縮メカニズムを深く理解します。Supermemory がコンテキストがオーバーフローする前に、構造化された要約を能動的に生成して永続的に保存し、長いセッションでの「記憶喪失」問題を防ぐ方法を学びます。

### [詳細設定の解説](./configuration/)

圧縮しきい値、キーワードトリガールール、検索パラメータをカスタマイズします。設定ファイルのすべてのオプションをマスターし、Supermemory をあなたのワークフローに完璧に適応させます。

</div>

## 学習パス

```
┌─────────────────────────────────────────────────────────────┐
│  ステップ 1             ステップ 2                             │
│  先制圧縮の原理   →   詳細設定の解説                           │
│  (メカニズムの理解)     (カスタマイズ実践)                     │
└─────────────────────────────────────────────────────────────┘
```

**推奨順序**：

1. **まず圧縮原理を学ぶ**：Supermemory がコンテキストを自動的に管理する方法を理解します。これは高度な設定の基礎です。
2. **次に設定の解説を学ぶ**：メカニズムを理解した後、合理的な設定決定（例：しきい値をいくつに設定するのが適切か）ができるようになります。
