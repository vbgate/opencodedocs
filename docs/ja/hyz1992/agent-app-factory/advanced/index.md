# 高度編集：パイプラインと内部メカニズム

本セクションでは、AI App Factory の中核メカニズムと高度な機能について詳しく解説します。7段階パイプラインの詳細な仕組み、Sisyphus スケジューラのスケジューリング戦略、権限とセキュリティメカニズム、失敗処理戦略、そしてコンテキスト最適化によりトークンコストを削減する方法について学びます。

::: warning 前提条件
本セクションを学習する前に、以下が完了していることを確認してください：
- [クイックスタート](../../start/getting-started/) と [インストール設定](../../start/installation/)
- [7段階パイプライン概要](../../start/pipeline-overview/)
- [プラットフォーム連携](../../platforms/claude-code/) 設定
:::

## セクション内容

本セクションには以下のトピックが含まれます：

### 7段階パイプライン詳解

- **[ステージ 1：Bootstrap - 構造化された製品アイデア](stage-bootstrap/)**
  - 曖昧な製品アイデアを構造化されたドキュメントに変換する方法を学ぶ
  - Bootstrap Agent の入出力形式を理解する

- **[ステージ 2：PRD - 製品要件ドキュメントの生成](stage-prd/)**
  - ユーザーストーリー、機能リスト、非目標を含む MVP レベルの PRD を生成する
  - 要件分解と優先順位付けのテクニックを習得する

- **[ステージ 3：UI - インターフェースとプロトタイプの設計](stage-ui/)**
  - ui-ux-pro-max スキルを使用して UI 構造とプレビュー可能なプロトタイプを設計する
  - インターフェース設計フローとベストプラクティスを理解する

- **[ステージ 4：Tech - 技術アーキテクチャの設計](stage-tech/)**
  - 最小限の実行可能な技術アーキテクチャと Prisma データモデルを設計する
  - 技術選定とアーキテクチャ設計の原則を習得する

- **[ステージ 5：Code - 実行可能なコードの生成](stage-code/)**
  - UI Schema と Tech 設計に基づいてフロントエンド・バックエンドコード、テスト、設定を生成する
  - コード生成フローとテンプレートシステムを理解する

- **[ステージ 6：Validation - コード品質の検証](stage-validation/)**
  - 依存関係のインストール、型チェック、Prisma スキーマ、コード品質を検証する
  - 自動化された品質チェックフローを習得する

- **[ステージ 7：Preview - デプロイガイドの生成](stage-preview/)**
  - 完全な実行説明ドキュメントとデプロイ設定を生成する
  - CI/CD 統合と Git Hooks 設定を学ぶ

### 内部メカニズム

- **[Sisyphus スケジューラ詳解](orchestrator/)**
  - スケジューラがパイプラインを調整、状態を管理、権限チェックを実行する仕組みを理解する
  - スケジューリング戦略とステートマシンの原理を習得する

- **[コンテキスト最適化：セッション分割実行](context-optimization/)**
  - `factory continue` コマンドを使用してトークンを節約する方法を学ぶ
  - 各ステージで新しいセッションを作成するベストプラクティスを習得する

- **[権限とセキュリティメカニズム](security-permissions/)**
  - 能力境界マトリクス、権限超過処理、セキュリティチェックメカニズムを理解する
  - セキュリティ設定と権限管理を習得する

- **[失敗処理とロールバック](failure-handling/)**
  - 失敗の識別、再試行メカニズム、ロールバック戦略、手動介入フローを学ぶ
  - トラブルシューティングと復旧テクニックを習得する

## 学習パス推奨

### 推奨学習順序

1. **まず7段階パイプラインを完了する**（順番に）
   - Bootstrap → PRD → UI → Tech → Code → Validation → Preview
   - 各ステージには明確な入出力があり、順番に学ぶことで完全な理解が得られる

2. **次にスケジューラとコンテキスト最適化を学ぶ**
   - Sisyphus がこの7ステージをどう調整するかを理解する
   - トークンコストを最適化する方法を学ぶ

3. **最後にセキュリティと失敗処理を学ぶ**
   - 権限境界とセキュリティメカニズムを習得する
   - 失敗シナリオと対応戦略を理解する

### 異なる役割の学習重点

| 役割 | 重点学習セクション |
| ---- | ---------------- |
| **開発者** | Code、Validation、Tech、Orchestrator |
| **プロダクトマネージャー** | Bootstrap、PRD、UI、Preview |
| **テックリード** | Tech、Code、Security、Failure Handling |
| **DevOps エンジニア** | Validation、Preview、Context Optimization |
