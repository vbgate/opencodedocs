# 付録：技術詳細と制限事項の説明

この章ではプラグインのリファレンスドキュメントと技術的な詳細を提供し、プラグインの設計思想、境界条件、パフォーマンス最適化戦略を深く理解できるようにします。

::: info 学習後の到達目標
- プラグインの既知の制限事項と適用可能なシナリオを理解する
- キャッシュメカニズムとパフォーマンス最適化戦略を習得する
- プラグインの技術的な境界と設計上のトレードオフを理解する
:::

## この章の内容

### 📚 [既知の制限事項：プラグインの境界](./limitations/)

プラグインがサポートしていない機能と技術的な制限を理解し、サポートされていないシナリオでの使用を避けます。以下を含みます：
- HTML テーブル、複数行セル、区切り行のないテーブルはサポートしていません
- セルの結合と設定オプションはサポートしていません
- 超大規模テーブルのパフォーマンスは検証されていません

**対象者**：プラグインでできること、できないことを知りたいユーザー

### 🔧 [技術詳細：キャッシュメカニズムとパフォーマンス最適化](./tech-details/)

プラグインの内部実装を深く理解します。キャッシュメカニズム、パフォーマンス最適化戦略、コード構造を含みます：
- widthCache データ構造とキャッシュ検索フロー
- 自動クリアメカニズムとキャッシュしきい値
- パフォーマンス最適化の効果分析

**対象者**：プラグインの実装原理に興味がある開発者

## 学習パスの推奨

この章の 2 つのサブページは比較的独立しているため、必要に応じて読むことができます：

1. **クイックスタートユーザー**：まず「既知の制限事項」を読むことをお勧めします。プラグインの境界を理解したらそこで停止できます
2. **深く学びたいユーザー**：順番に読む → 「既知の制限事項」→「技術詳細」
3. **開発者**：完全な読了をお勧めします。プラグインの設計と将来の拡張を理解するのに役立ちます
