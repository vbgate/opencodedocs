# デプロイメントオプション

## 学習目標

本レッスンを完了すると、以下のことができるようになります：

- ニーズに最適なデプロイメント方式を選択（ローカル、Docker、Nix、クラウドサービス）
- ローカル環境にClawdbotをインストールして実行
- Dockerを使用してGatewayをコンテナ化デプロイ
- Nixで宣言型にClawdbotを管理
- GatewayをFly.io、Hetzner VPS、exe.devにデプロイ
- リモートアクセスとセキュリティベストプラクティスを設定

## 現在の課題

Clawdbotを使用したいが、どのデプロイメント方式を選択すべきか迷っています：

- ローカルインストールは最も簡単だが、PCの電源を切ると使用できない
- クラウドで24時間365日稼働させたいが、どのクラウドサービスが適切か分からない
- 設定ミスを恐れており、最も確実なデプロイメント案を探している
- 複数デバイスから同じGatewayにアクセスする必要があるが、その実現方法が分からない

## デプロイ方式の活用シナリオ

| デプロイ方式 | 適用シナリオ |
|---|---|
| **ローカルインストール** | 個人PC、開発テスト、クイックスタート |
| **Docker** | 環境分離、サーバーデプロイ、高速再構築 |
| **Nix** | 再現性のあるデプロイ、既存のNixOS/Home Managerユーザー、バージョンのロールバックが必要な場合 |
| **Fly.io** | クラウド24時間365日稼働、自動HTTPS、簡略化された運用 |
| **Hetzner VPS** | 独自のVPS、完全な制御、低コスト24時間365日稼働 |
| **exe.dev** | 安価なLinuxホスト、VPSの自己設定が不要 |

## 準備事項

開始前に、以下を確認してください：

::: warning 環境要件
- Node.js ≥ 22（ローカルインストールに必須）
- Docker Desktop + Docker Compose v2（Dockerデプロイに必須）
- Nix flakes + Home Manager（Nixデプロイに必須）
- SSHクライアント（クラウドデプロイアクセスに必須）
- 基本的なターミナル操作能力（コピー、ペースト、コマンド実行）
:::

::: tip 推奨ツール
  - Clawdbotを初めて使用する場合は、[クイックスタート](../../start/getting-started/)から開始することをお勧めします
- AIアシスタント（Claude、Cursorなど）の使用により、設定プロセスを加速できます
- API Key（Anthropic、OpenAIなど）とチャネル認証情報を保存してください
:::

## デプロイ方式の比較

### ローカルインストール

**適用**：個人PC、開発テスト、クイックスタート

**メリット**：
- ✅ 最もシンプルで直接、追加インフラストラクチャ不要
- ✅ 起動が高速、デバッグが容易
- ✅ 設定変更は即時反映

**デメリット**：
- ❌ PCの電源を切ると使用不可
- ❌ 24時間365日サービスには不向き
- ❌ 複数デバイス同期に追加設定が必要

### Dockerデプロイ

**適用**：サーバーデプロイ、環境分離、CI/CD

**メリット**：
- ✅ 環境分離、簡単にクリーンアップと再構築
- ✅ クロスプラットフォーム統一デプロイ
- ✅ サンドボックス分離ツール実行をサポート
- ✅ 設定をバージョン管理可能

**デメリット**：
- ❌ Dockerの知識が必要
- ❌ 初回設定が少し複雑

### Nixデプロイ

**適用**：NixOSユーザー、Home Managerユーザー、再現性のあるデプロイが必要

**メリット**：
- ✅ 宣言型設定、再現性
- ✅ 高速ロールバック（`home-manager switch --rollback`）
- ✅ 全コンポーネントバージョン固定
- ✅ Gateway + macOS app + ツール全管理

**デメリット**：
- ❌ 学習曲線が急
- ❌ Nixエコシステムの理解が必要

### クラウドデプロイ（Fly.io / Hetzner / exe.dev）

**適用**：24/7オンライン、リモートアクセス、チーム協働

**メリット**：
- ✅ 24/7オンライン、ローカルマシンに依存しない
- ✅ 自動HTTPS、手動証明書不要
- ✅ 高速スケール
- ✅ 複数デバイスリモートアクセスをサポート

**デメリット**：
- ❌ クラウドサービス費用が必要
- ❌ 基本的な運用知識が必要
- ❌ データはサードパーティに保存
