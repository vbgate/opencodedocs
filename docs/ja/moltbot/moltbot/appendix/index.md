# 付録

本章では Clawdbot の高度なリファレンスドキュメントと開発リソースを提供します。完全な設定リファレンス、Gateway WebSocket API プロトコル仕様、デプロイオプション、開発ガイドが含まれます。

::: info 対象シナリオ
本章は Clawdbot の内部メカニズムを深く理解したい、高度な設定を行いたい、デプロイや開発に参加したいユーザーに適しています。初めて使用する場合は、先に[クイックスタート](../../start/getting-started/)章を完了することをお勧めします。
:::

## サブページナビゲーション

### [完全設定リファレンス](./config-reference/)
**詳細設定ファイルリファレンス** - すべての設定項目、デフォルト値、例を網羅。Gateway、Agent、チャンネル、ツールなどのモジュールの完全な設定説明を確認できます。

### [Gateway WebSocket API プロトコル](./api-protocol/)
**プロトコル仕様ドキュメント** - Gateway WebSocket プロトコルの完全仕様。エンドポイント定義、メッセージ形式、認証方式、イベント購読メカニズムを含みます。カスタムクライアントの作成や Gateway との統合を行う開発者に適しています。

### [デプロイオプション](./deployment/)
**デプロイ方法ガイド** - 異なるプラットフォームでのデプロイ方法：ローカルインストール、Docker、VPS、Fly.io、Nix など。さまざまな環境で Clawdbot を実行する方法を学びます。

### [開発ガイド](./development/)
**開発者ドキュメント** - ソースコードからのビルド、プラグイン開発、テスト、貢献のフロー。Clawdbot プロジェクトの開発に参加する方法や、カスタムプラグインを記述する方法を学びます。

## 学習パスの推奨

ニーズに合わせて、適切な学習パスを選択してください。

### 設定・運用担当者
1. まず[完全設定リファレンス](./config-reference/)を読む - すべての設定可能項目を理解
2. [デプロイオプション](./deployment/)を参照 - 適切なデプロイソリューションを選択
3. 必要に応じて Gateway WebSocket API ドキュメントを参照して統合

### アプリケーション開発者
1. [Gateway WebSocket API プロトコル](./api-protocol/)を読む - プロトコルメカニズムを理解
2. [完全設定リファレンス](./config-reference/)を確認 - 関連機能の設定方法を理解
3. プロトコルの例を参照してクライアントを構築

### プラグイン・機能開発者
1. [開発ガイド](./development/)を読む - 開発環境とビルドフローを理解
2. [Gateway WebSocket API プロトコル](./api-protocol/)を深く学ぶ - Gateway アーキテクチャを理解
3. [完全設定リファレンス](./config-reference/)を参照 - 設定システムを理解

## 前提条件

::: warning 前提知識
本章に深く入る前に、以下を完了することをお勧めします：
- ✅ [クイックスタート](../../start/getting-started/)を完了
- ✅ 少なくとも1つのチャンネル（[WhatsApp](../../platforms/whatsapp/) や [Telegram](../../platforms/telegram/) など）を設定
- ✅ 基本的な AI モデル設定を理解（[AI モデルと認証](../../advanced/models-auth/)参照）
- ✅ JSON 設定ファイルと TypeScript の基本知識
:::

## 次のステップ

本章の学習を完了すると、以下が可能になります：

- **高度な設定を行う** - [完全設定リファレンス](./config-reference/)を参照して Clawdbot をカスタマイズ
- **本番環境にデプロイ** - [デプロイオプション](./deployment/)に従って適切なデプロイソリューションを選択
- **カスタム機能を開発** - [開発ガイド](./development/)を参照してプラグインを記述したりコードを貢献したり
- **他の機能を深く学ぶ** - [高度な機能](../../advanced/)章を探索。セッション管理、ツールシステムなど

::: tip ヘルプの探し方
使用中に問題が発生した場合、[トラブルシューティング](../../faq/troubleshooting/)を参照して一般的な問題の解決策を確認できます。
:::
