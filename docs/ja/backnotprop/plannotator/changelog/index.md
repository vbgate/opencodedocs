# 更新履歴

この章では Plannotator のバージョン履歴、新機能リリース、バグ修正情報を記録し、プロジェクトの進化と最新動向を理解できるようにします。

## 本章の内容

<div class="grid-cards">

<a href="./release-notes/" class="card">
  <h3>📋 バージョン履歴</h3>
  <p>Plannotator の完全なバージョン履歴を確認します。各バージョンの新機能、改善、バグ修正が含まれます。コードレビュー、画像アノテーション、Obsidian/Bear 連携などの主要機能のリリースタイムラインを把握できます。</p>
</a>

</div>

## 読む際のヒント

更新履歴は以下のシーンで読むことをお勧めします。

| シーン | お勧め |
|--- | ---|
| **アップグレード前の確認** | 対象バージョンの新機能と破壊的変更を確認する |
| **機能探索** | 最新バージョンでどのような機能が追加されたかを理解する |
| **トラブルシューティング** | 特定のバグが新しいバージョンで修正されているかを確認する |
| **バージョン互換性** | Plannotator と Claude Code/OpenCode のバージョン対応関係を確認する |

::: tip 最新版を維持する
定期的に更新履歴を確認し、新機能とセキュリティ修正を取得するために最新バージョンにアップグレードすることをお勧めします。
:::
