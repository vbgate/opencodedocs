# 高級機能

プランレビューとコードレビューの基本をマスターしたら、このセクションでPlannotatorの上級機能を解放しましょう：チームコラボレーション共有、ノートアプリ連携、リモート開発サポート、そして柔軟な環境変数設定について学びます。

::: warning 前提条件
このセクションを学ぶ前に、以下を完了していることを確認してください：
- [クイックスタート](../start/getting-started/)：Plannotatorの基本概念を理解
- [プランレビューの基本](../platforms/plan-review-basics/)：基本的なプランレビュー操作をマスター
:::

## このセクションの内容

| レッスン | 説明 | 適したシナリオ |
|--- | --- | ---|
| [URL共有](./url-sharing/) | URLでプランとアノテーションを共有し、バックエンド不要のチームコラボレーションを実現 | チームメンバーとレビュー結果を共有したい |
| [Obsidian連携](./obsidian-integration/) | 承認されたプランをObsidian vaultに自動保存 | Obsidianでナレッジベースを管理している |
| [Bear連携](./bear-integration/) | x-callback-urlでプランをBearに保存 | macOS/iOSユーザーでBearノートを使用 |
| [リモートモード](./remote-mode/) | SSH、devcontainer、WSLなどのリモート環境で使用 | リモート開発シナリオ |
| [環境変数設定](./environment-variables/) | 利用可能なすべての環境変数とその用途を理解 | Plannotatorの動作をカスタマイズしたい |

## 推奨学習パス

```
┌─────────────────────────────────────────────────────────────────┐
│  推奨学習順序                                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. URL共有            ← 最もよく使う、チームコラボに必須           │
│       ↓                                                         │
│  2. ノートアプリ連携     ← 使用しているノートアプリに応じて選択       │
│     ├─ Obsidian連携     （Obsidianユーザー）                      │
│     └─ Bear連携         （Bearユーザー）                          │
│       ↓                                                         │
│  3. リモートモード       ← リモート環境で開発している場合            │
│       ↓                                                         │
│  4. 環境変数設定         ← 詳細なカスタマイズが必要な場合に参照      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

::: tip 必要に応じて学習
すべてのコンテンツを順番に学ぶ必要はありません。共有機能だけが必要なら、URL共有を学べば十分です。リモート開発を使用しないなら、リモートモードはスキップできます。
:::
