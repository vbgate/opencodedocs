# FAQと最適化

この章節は、Clawdbotを使用する際に遭遇する様々な問題を解決し、完全なコマンドライン操作方法をマスターし、システムパフォーマンスを最適化する方法を学ぶのに役立ちます。

::: info 学習順序の推奨
この章節の内容を以下の順序で読むことをお勧めします：
1. まず **[CLIコマンドリファレンス](./cli-commands/)** を学ぶ —— 基本的な操作とデバッグツールをマスター
2. 次に **[トラブルシューティング](./troubleshooting/)** を学ぶ —— 問題発生時に迅速に特定と解決が可能
3. 最後に **[パフォーマンス最適化](./performance-optimization/)** を学ぶ —— システム実行効率を向上
:::

## サブページナビゲーション

### [トラブルシューティング](./troubleshooting/)

一般的な問題を解決する包括的なトラブルシューティングガイド、以下を網羅：
- Gateway起動失敗
- チャンネル接続問題（WhatsApp、Telegram、Slackなど）
- 認証エラーと権限問題
- ツール呼び出し失敗
- セッション管理異常

**使用シナリオ**：具体的な問題に遭遇し、迅速にトラブルシューティングと解決が必要な場合。

---

### [CLIコマンドリファレンス](./cli-commands/)

完全なコマンドリストと使用法、以下を含む：
- Gateway制御コマンド（開始、再起動、状態）
- Agent操作コマンド
- モデル管理と設定
- チャンネル管理（ログイン、状態、ログアウト）
- セッション管理（リスト、リセット、削除、圧縮）
- ノード管理（リスト、呼び出し、ペアリング）
- スキル管理と設定ウィザード

**使用シナリオ**：特定のコマンドの使用法とパラメータを迅速に検索する場合、またはCLIの完全な機能を学ぶ場合。

---

### [パフォーマンス最適化](./performance-optimization/)

Gatewayパフォーマンスと応答速度を向上させる最適化技術、以下を網羅：
- プロンプトキャッシュ（Prompt Caching）によるコスト削減
- セッションプルーニング（Session Pruning）によるコンテキスト削減
- 同時実行制御とリソース管理
- メモリ最適化とガベージコレクション
- ログレベルチューニング

**使用シナリオ**：システム実行が遅い、リソース使用率が高い、またはユーザー体験を向上させたい場合。
